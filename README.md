# Novel Master

面向**长篇小说创作**与 **AI Agent 协作**的写作工具。把设定、章节、提示词等放进统一的虚拟文件树里，在「项目 → 会话」中一边改稿一边对话，Agent 通过 `vfs.*` 工具直接读写工作区文件。

当前交付：**Android 移动端** + **`nm` 命令行** + 共享 **Core** 领域层（CLI / App 行为一致）。

## 它能做什么

### 项目与会话

- **项目**：一部作品或一个创作宇宙；可维护**项目模板**（默认文件树、目录规则）。
- **会话**：项目下的单次写作/对话线；新建会话时从项目模板复制工作区，之后与会话内改稿互不影响。
- **消息**：会话内多轮对话（用户 / 助手 / 系统），支持流式输出与消息管理。

### 三层工作区（VFS）

所有文件在逻辑上都以 **`/` 为根**（例如 `/notes/设定.md`、`/chapters/01.md`），路径在全局模板、项目模板、会话工作区中**形状一致**，便于备份、迁移和 Agent 理解。

| 范围 | 用途 |
|------|------|
| **全局模板** | 跨项目共用的种子内容（世界观、通用提示词等） |
| **项目模板** | 某一作品默认自带的目录与文件 |
| **会话工作区** | 当前会话正在编辑的文件树；Agent 主要在这里读写 |

移动端在 **「会话」** 与 **「项目模板」** 入口分别管理；会话内还有 **「会话工作区」** 文件页签。

### 文件管理

- 目录浏览、新建文件/文件夹、重命名、删除
- **从上级同步**：项目模板 ← 全局；会话工作区 ← 项目模板（覆盖式拉取）
- **ZIP 导出 / 导入**：按域打包纯文本快照；导入即**该域全量替换**（适合备份与换机恢复）
- 文本文件编辑（含 Markdown 预览）

### Worktree（展示规则）

在不改文件内容的前提下，控制「哪些文件参与列表、如何排序、如何拼进 Agent 上下文」：

- 目录级规则：排序字段、头尾条数、填充策略、规则开关
- 文件级：**自动 / 展示 / 隐藏**（跟随、关闭等状态在 UI 中可见）
- 根目录规则不可关闭，保证工作区始终可导航

### Agent

- 内置 **VFS 工具**（读、写、替换、列表、mkdir、glob、grep 等），路径与会话工作区一致
- **Agent 定义**可配置 **工具白名单或黑名单**（二选一），便于做「只读分析」或「全权写作」类角色
- 写作相关变更可走 **Session FS** 批次执行与回滚（版本校验可配置）

### 配置与扩展（部分在 App / CLI）

- 模型与 Provider、Agent 列表与编辑器
- 正则规则、压缩策略等（随迭代逐步完善）

## 典型使用流程

1. 在**全局模板**里放好通用设定与提示词骨架。  
2. **创建项目**，编辑**项目模板**（卷纲、人设、章节占位等）。  
3. **新建会话**，在**会话工作区**写正文，与 Agent 对话改稿。  
4. 需要时 **导出 ZIP** 备份，或 **从上级同步** 拉最新模板。  
5. 进阶用户用 **`nm`** 脚本化批量读写、导入导出、调试 Worktree。
