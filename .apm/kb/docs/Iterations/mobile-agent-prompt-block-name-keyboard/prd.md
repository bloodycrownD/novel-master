# Mobile Agent Prompt 块名称键盘失焦 Bugfix PRD

> **类型**：缺陷修复（bugfix）  
> **平台**：Android（真机 / 模拟器）  
> **路径**：`apps/mobile` → Agent 编辑页 →「Prompt 块」→ 添加或编辑块 →「名称」字段

## 背景

在 Android 上编辑 Agent 的 Prompt 块时，用户在「名称」输入框中无法正常输入：每按一次软键盘上的任意字符键（字母、数字、符号等），输入框会立刻失焦并退出输入模式，键盘收起，导致名称几乎无法编辑。该问题使 Prompt 块的创建与配置在移动端不可用，属于阻断性体验缺陷。

相关界面位于 Agent 编辑表单的 Prompt 块卡片内「名称」字段（`AgentEditorForm`）。

## 目标（含成功指标）

| 目标 | 成功指标 |
|------|----------|
| 名称可编辑 | Android 上可在 Prompt 块「名称」字段连续输入完整字符串（≥20 个字符抽测），期间输入框保持聚焦、软键盘保持打开 |
| 主动结束编辑 | 用户点击页面其他区域或切换焦点前，不会因单次按键自动失焦 |
| Agent 表单回归 | Agent 编辑页全部可编辑字段手工抽测通过，无新增失焦/无法输入类问题 |
| 不扩功能面 | 仅修复输入体验，不改变 Prompt 块数据模型、保存语义或 UI 结构 |

## 用户与场景

| 用户 | 场景 |
|------|------|
| 在手机上配置 Agent 的用户 | 新建文本块/会话块后，将默认名 `block-N` 改为有意义的名称（如 `system-rules`、`history`） |
| 在手机上维护 Agent 的用户 | 编辑已有 Prompt 块，修正块名称以便区分多块配置 |

## 范围

### 包含范围

1. 修复 Android 上 Agent 编辑页 Prompt 块「名称」字段：单次按键即失焦、键盘收起的问题。
2. 对 Agent 编辑页全部可编辑字段做回归验收（名称、Agent 基础信息、采样参数、文本块角色/内容等），确认修复未引入新的输入异常。
3. 在 Android 真机或官方模拟器上完成验收记录（机型/系统版本可记在测试备注中，非 PRD 强制字段）。

### 不包含范围

- iOS、Expo Web、移动浏览器（本期未报告；若后续复现可单独立项）。
- Web/桌面端 Agent 编辑器、CLI、`examples/mobile` 原型同步。
- Prompt 块功能新增（块类型、排序规则、校验规则、默认命名策略变更等）。
- 技术方案、接口设计、组件重构说明（另文处理，本 PRD 不涉及）。

## 核心需求

1. **连续输入**：在 Prompt 块「名称」`TextInput` 中，用户连续按键时，焦点保持在当前输入框，软键盘不因单次字符输入而关闭。
2. **内容可保存**：用户输入的名称应通过现有 `onChangeText` / 保存流程正确反映到块配置，保存后再次进入页面显示一致。
3. **不破坏同页其它输入**：修复后 Agent 编辑页其它 `FormTextInput`、`PromptMacroTextInput`、`FormSelectField` 等等交互与修复前一致或更好，不得出现新的「按一键失焦」现象。
4. **添加与编辑均覆盖**：无论通过「添加」新建块还是编辑列表中已有块，「名称」字段行为一致。
5. **最小改动原则**：以修复失焦根因为目标，避免无关 UI 或业务逻辑变更。

## 验收标准

### Prompt 块名称（主路径）

- **Given** 用户在 Android 上打开任意 Agent 的编辑页，且至少有一个 Prompt 块（或刚通过「添加」新建一块）  
- **When** 用户聚焦该块的「名称」字段，并使用系统默认输入法连续输入 `my-prompt-block-01`（含字母、数字、连字符）  
- **Then** 输入过程中输入框始终聚焦、软键盘保持打开；输入结束后字段显示完整字符串，无字符丢失或中途自动失焦

- **Given** 用户已在「名称」字段输入部分内容  
- **When** 用户点击同页非输入区域（如块卡片空白处、其它只读文案）  
- **Then** 输入框可按预期失焦、键盘收起（主动结束编辑）

### Agent 编辑页回归

- **Given** 用户在 Android 上打开同一 Agent 编辑页  
- **When** 依次抽测以下可编辑项（每项至少输入/修改 3 次以上）：Agent 名称、模型相关文本字段、maxSteps、Prompt 块「名称」、文本块「内容」（含宏输入若可用）、采样相关数字/文本字段（若页面展示）  
- **Then** 均无「按任意字符键一次即失焦」现象；保存后重新进入页面，修改仍正确持久化

### 负向（修复前现象不得复现）

- **When** 用户在 Prompt 块「名称」字段按下软键盘任意字符键一次  
- **Then** 不得出现：输入框失焦、软键盘立即收起、已输入字符未显示即中断（修复前缺陷）
