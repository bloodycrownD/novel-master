# file-inclusion-status-modal Bug PRD

## 背景

Worktree 对每个**文件**有三种**纳入方式**（是否进入 Agent 上下文），Core 与 Mobile 产品口径为 **展示 / 隐藏 / 跟随**。Desktop 早期将三者拆成右键菜单三个平铺项，且将 `auto` 误标为「跟随目录」，用户难以理解，也不符合「一处设置、三选一」的心智。

## 现象描述

- 文件右键菜单同时出现「隐藏文件」「展示文件」「跟随目录」三项，语义重叠、菜单过长
- 「跟随目录」名称易误解为目录级操作；实际为文件纳入方式 **跟随**（由父目录规则决定）
- 用户期望：单一「状态设置」入口，弹窗内 **展示 / 隐藏 / 跟随** 三单选

## 复现步骤

1. Desktop 进入会话，Explorer 中右键任意 **文件**
2. 查看上下文菜单

## 预期行为

- 菜单含 **「状态设置」**（及重命名、删除等）
- 点击后弹出对话框，三个 **单选**：**展示**、**隐藏**、**跟随**，当前状态预选
- 保存后文件行右侧状态（如「跟随 · 不展示」）更新

## 实际行为（修复前）

- 三个平铺菜单项；「跟随目录」文案不准确
- 无弹窗集中设置

## 影响范围

- Desktop 工作区文件右键菜单
- 不影响目录右键（目录仍为「规则配置」等）
- 不影响 Core `inclusionMode` 语义与 IPC

## 纳入方式说明（产品）

| 选项 | 含义 |
|------|------|
| **跟随** | 默认；是否纳入 Agent 上下文由**父目录纳入规则**计算（可与「显示状态」组合，如「跟随 · 不展示」） |
| **展示** | 强制纳入，显示全内容 |
| **隐藏** | 强制不纳入 |

## 验收标准

- **Given** 某文件当前为「跟随 · 不展示」  
  **When** 右键 →「状态设置」  
  **Then** 弹窗打开且 **跟随** 为选中态

- **Given** 弹窗打开  
  **When** 选择「展示」并保存  
  **Then** 文件行状态变为「展示 · 全内容」（或等价展示态），菜单中 **无**「跟随目录」等旧项

- **Given** 文件右键菜单  
  **When** 查看项列表  
  **Then** **无**「隐藏文件/展示文件/跟随目录」平铺三项

## 回归测试要点

- 目录右键、新建/删除/重命名不受影响
- `ipcWorktreeSetFileRule` 行为与改前一致
