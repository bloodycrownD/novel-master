# Mobile Agent 导航与当前 Agent 设置 PRD

> **类型**：体验重构 / 功能调整  
> **平台**：Android（真机 / 模拟器）  
> **范围**：`apps/mobile` 底部导航、我的页、Agent 列表、会话操作抽屉

## 背景

当前 App 将 **Agent** 作为底部独立 Tab，列表支持「设为默认」并在卡片上展示「默认」徽标；工作区与会话内切换 Agent 的入口分散，与「当前模型」的统一体验不一致。

用户希望：**收敛 Agent 管理入口**到「我的」，用**工作区「当前 agent」**替代「默认 Agent」概念，并在对话中提供与「切换模型」对等的**「切换 agent」**快捷入口，使 Agent 选择与模型选择遵循同一套心智模型。

## 目标（含成功指标）

| 目标 | 成功指标 |
|------|----------|
| 简化主导航 | 底部 Tab 仅保留 **「对话」「我的」**；无 Agent Tab |
| 统一 Agent 管理入口 | 「我的 → 配置 → agent设置」进入原 Agent Tab 同等能力（列表、新建、编辑、复制、删除、批量管理） |
| 统一当前 Agent 语义 | 移除「设为默认」与列表「默认」徽标；**当前 agent** 由工作区设置，持久化至 `PersistentState.currentAgentId` |
| 对齐当前模型体验 | 「我的 → 工作区 → 当前 agent」交互与「当前模型」一致（展示名称、点击弹出选择器） |
| 对话内快捷切换 | 会话操作抽屉增加 **「切换 agent」**，与「切换模型」并列；抽屉顶部展示**当前 agent** 只读信息（对齐「当前模型」区块） |
| 可重命名 | Agent 列表 ⋮ 菜单含 **「重命名」**，修改 `definition.name`（显示名） |
| 行为一致 | 切换当前 agent 后，新发送消息使用新 agent；顶栏/元信息展示的 agent 名与所选一致 |

## 用户与场景

| 用户 | 场景 |
|------|------|
| 日常写作用户 | 在「我的」设置默认使用的 Agent，无需理解「设为默认」与 Tab 跳转 |
| 多 Agent 用户 | 在对话中通过抽屉快速切换 Agent，类似切换模型 |
| 维护 Agent 用户 | 在「agent设置」中新建、编辑、复制、删除、**重命名** Agent |

## 范围

### 包含范围

1. **移除底部 Agent Tab**；`RootNavigator` 主 Tab 仅「对话」「我的」。
2. **「我的 → 配置」** 新增入口 **「agent设置」**（与「服务商管理」等同级），点击进入 **Stack 全屏子页**，内容为原 Agent Tab 页面（`AgentList`：管理、新建、进编辑器、批量删除等）。
3. **Agent 列表菜单调整**：
   - **移除**「设为默认」；
   - **移除** 卡片「默认」徽标；
   - **新增**「重命名」（改 Agent **显示名称** `definition.name`）；
   - 保留「复制」「删除」及现有批量能力。
4. **「我的 → 工作区」** 新增 **「当前 agent」** 行：展示当前 agent 显示名；点击打开 **Agent 选择器**（交互对齐「当前模型」+ `ModelPickerModal`）；选中后写入 `PersistentState.setCurrentAgentId`。
5. **会话操作抽屉**（聊天页 ☰）：
   - 顶部只读区增加 **「当前 agent」**（与「当前模型」并列或同等样式）；
   - 操作列表增加 **「切换 agent」**（与「切换模型」并列），打开同一 Agent 选择器并更新当前 agent。
6. **未设置 currentAgentId 时**：沿用现有 Core/App 回退逻辑（如 registry 首项），列表/顶栏展示与 run 行为一致。
7. Android 手工验收（见下文）。

### 不包含范围

- iOS（本期未指定；实现后若需对齐可单独立项）。
- Agent **ID**（registry 键）重命名、YAML 内 `agents` 键名变更。
- Agent 编辑器表单结构、Prompt 块、工具策略等业务变更。
- CLI / Desktop / `examples/mobile` 原型同步。
- 新增「默认 Agent」以外的排序、分组、搜索。
- 技术方案、接口设计、组件拆分（见 SPEC）。

## 核心需求

1. **两 Tab 主导航**：用户打开 App 仅见「对话」「我的」；原 Agent Tab 能力无损迁移至「agent设置」。
2. **配置区入口**：「agent设置」位于 **「我的 → 配置」** 区块，进入后为全屏子页，带系统/顶栏返回。
3. **废弃「默认」产品概念**：UI 不再出现「设为默认」「默认」徽标；**当前 agent** 即工作区指针，与会话 run、顶栏展示一致。
4. **工作区当前 agent**：与「当前模型」同级 UX——可见当前值、可点击更换、更换后立即刷新展示。
5. **会话内切换 agent**：抽屉中「切换 agent」一步到达选择器；切换后关闭抽屉，后续 run 使用新 agent。
6. **重命名 Agent**：通过 ⋮ →「重命名」修改显示名；校验非空；保存后列表与「当前 agent」展示同步更新。
7. **删除当前 agent 时的指针**：删除列表中含当前 agent 时，行为与现有一致（切至剩余首项或 reset），且不 crash。

## 验收标准

### 底部导航

- **Given** 用户打开 App  
- **When** 查看底部 Tab  
- **Then** 仅有「对话」「我的」；**无** Agent Tab

### agent设置（原 Tab 能力）

- **Given** 用户在「我的 → 配置」  
- **When** 点击「agent设置」  
- **Then** 进入全屏 Agent 列表页，可「新建」「管理」、点卡片进编辑器、⋮ 菜单含复制/删除/重命名；**无**「设为默认」

### 列表与徽标

- **Given** Agent 列表有多条  
- **When** 浏览列表  
- **Then** **无**「默认」徽标；当前 agent 不在列表上用徽标强调（当前值仅在「我的→当前 agent」与会话抽屉展示）

### 重命名

- **Given** 某 Agent 显示名为 `new-agent`  
- **When** ⋮ →「重命名」→ 输入 `writer` 并确认  
- **Then** 列表显示 `writer`；若其为当前 agent，「我的→当前 agent」与会话抽屉只读区同步为 `writer`

### 工作区当前 agent

- **Given** 用户在「我的 → 工作区」  
- **When** 查看「当前 agent」行  
- **Then** 展示当前 agent 显示名（或未设置时的合理占位，与模型「—」风格一致）

- **Given** 用户点击「当前 agent」  
- **When** 在选择器中选择另一 Agent  
- **Then** 行内展示更新；`PersistentState.currentAgentId` 已更新；返回对话 Tab 后 run/顶栏 agent 名与新选择一致

### 会话抽屉切换 agent

- **Given** 用户在对话页打开会话操作抽屉  
- **When** 查看顶部信息区  
- **Then** 可见 **当前 agent** 与 **当前模型**（只读）

- **Given** 同上  
- **When** 点击「切换 agent」并选择另一 Agent  
- **Then** 抽屉关闭；当前 agent 更新；再次打开抽屉显示新 agent 名；下一条消息 run 使用新 agent

### 负向

- **When** 用户在 Agent 列表 ⋮ 菜单  
- **Then** **不得**出现「设为默认」

- **When** 用户从底部导航寻找 Agent Tab  
- **Then** **不得**存在该 Tab

### 回归（抽测）

- **Given** 用户完成：agent设置新建 → 工作区设当前 → 对话发送一条 → 抽屉切换 agent → 再发送  
- **When** 全流程完成  
- **Then** 无 crash、无 Tab 丢失；Agent 编辑/复制/删除仍可用
