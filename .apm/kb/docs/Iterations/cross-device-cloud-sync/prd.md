# Desktop / Mobile 跨端云同步 PRD

## 背景

Novel Master 的业务数据集中在各端本地的 `novel.db`（项目、会话、消息、Agent、工作区 VFS、偏好等）。Desktop 与 Mobile 已通过 **手动导出/导入 `.nmbackup`** 实现换机迁移，语义为整库替换；`import-export-navigation-fix` 迭代进一步约定：**导出不含服务商三表，导入保留本机服务商与 API Key**（SKSP 与平台绑定，跨端密文无法通用）。

历史 PRD（如 `mobile-app`、`chat-project-vfs`）将 **云同步、多设备冲突自动合并** 明确列为不包含范围。随着 Desktop 与 Mobile 双端并存，用户需要在 **手机与电脑之间交替创作** 时，比「U 盘式手动传文件」更轻量的互通方式，但一期不追求实时双向同步或实体级合并。

本需求在既有 **数据库备份能力** 之上，增加 **用户自备对象存储（S3 兼容 / OSS）** 作为远端载体，通过 **手动拉取（Pull）/ 推送（Push）** 与 **推送时租约锁** 降低双端互相覆盖的风险。

## 目标（含成功指标）

| 目标 | 成功指标 |
|------|----------|
| 双端数据可经云端互通（除服务商外） | 在配置同一存储桶后，一端 Push、另一端 Pull，可恢复项目/会话/消息/VFS 等核心数据 |
| 推送互斥、避免并发覆盖 | 同一时刻仅一台设备 Push 成功；另一台 Push 失败时得到明确提示，**0 次**静默覆盖对方刚推送的数据 |
| 云端有更新时先拉后推 | 当云端版本高于本机已同步版本时，Push 前 **100%** 阻断或强提示要求先 Pull |
| 持锁异常可自动恢复 | 设备 Push 中断后，在约定租约时间内其他设备无法 Push；租约过期后另一台可正常 Push，**无需**人工联系客服 |
| 服务商仍各端独立 | Pull/Push 后本机已配置的服务商与 API Key **仍可正常调用** |
| 双端体验一致 | Mobile（Android）与 Desktop 均提供配置入口与 Pull/Push 操作，核心文案与行为一致 |

**整体验收**：上述指标在「Desktop + Android Mobile、同一 S3 兼容桶」的手工用例中全部通过；不破坏现有本地导出/导入 `.nmbackup` 能力。

## 用户与场景

| 用户 | 场景 |
|------|------|
| 双端创作者 | 在家 Desktop 写作，外出用手机续写同一项目；回家前在手机上 **Push**，回家后 **Pull** 继续 |
| 换机/重装用户 | 新设备安装 App 并配置云存储后 **Pull** 一次即可恢复创作数据，无需找旧机导文件 |
| 谨慎型用户 | 仅使用 **Pull** 从云端恢复；仅在确认本机改动需要上传时手动 **Push** |
| 多设备家庭用户 | 两人共用同一存储桶时，后 Push 者若对方正在推送，收到「另一台设备正在同步」类提示而非静默失败 |

## 范围

### 包含范围

1. **云存储配置（用户自备）**  
   - 支持 **S3 兼容 API** 的对象存储（含自建 MinIO、主流云厂商 OSS 等）。  
   - 用户填写：Endpoint、Bucket、Access Key、Secret Key（及必要时的 Region / 路径前缀）。  
   - 提供 **测试连接**，成功后可在本机记住配置（密钥经各端既有安全存储保护，如 SKSP / 平台凭据库）。  
   - 每端生成并持久化 **设备标识**，用于锁的持有方展示。

2. **远端协调数据（业务语义）**  
   - 桶内维护 **同步状态**（版本号、当前快照位置、编辑锁与过期时间）。  
   - 快照文件为 **`.nmbackup` 语义** 的数据库备份（**不含服务商三表**，与本地导出一致）。  
   - 单调递增的 **版本号（rev）**：每次 Push 成功 +1，供双端判断是否需要 Pull。

3. **手动 Pull（拉取）**  
   - 入口：Mobile「我的 → 数据管理」、Desktop「设置 → 数据管理」（与现有导入/导出同级或同区）。  
   - 读取远端版本；若有可用快照且用户确认，则下载并 **导入**（保留本机服务商；导入前自动保留本地可恢复备份，与现有导入一致）。  
   - Pull **不需要** 持有编辑锁。  
   - Pull 成功后更新本机「已同步版本号」。

4. **手动 Push（推送）**  
   - 用户显式触发；**Agent 运行中禁止** Push（与现有导出一致）。  
   - Push 前：若 **云端版本高于本机已同步版本**，须先 Pull 或经用户明确确认「仍要覆盖云端」（默认阻断）。  
   - Push 时 **自动申请编辑锁**（带过期时间的租约）；申请失败则中止并提示原因。  
   - Push 成功：上传新快照、版本号 +1、**自动释放编辑锁**。  
   - Push 失败：尝试释放编辑锁；若未能释放，依赖 **锁过期** 使其他设备可再次 Push。

5. **编辑锁与超时**  
   - 锁仅在 **Push 过程** 持有，不在打开应用或日常编辑时长期占锁。  
   - 锁带 **过期时间**；过期后视为无人持锁，其他设备可 Push。  
   - 读取锁时：未过期且持有方非本机 → 视为「另一台设备正在推送或刚推送」，本机 Push 失败并提示。  
   - 上传耗时较长时，允许在 Push 过程中 **延长租约**（避免大库上传中途锁过期）。

6. **状态展示**  
   - 展示：是否已配置云存储、云端当前版本号、本机已同步版本号、上次 Pull/Push 时间与结果。  
   - 云端较新时给出「建议先拉取」提示（进入 Push 前或数据管理页）。

7. **平台**  
   - **Desktop + Android Mobile** 验收。  
   - 与现有 **本地** 导出/导入 `.nmbackup` **并存**，不删除原能力。

### 不包含范围

- **自建同步服务器**、账号体系、官方托管云空间。  
- **后台自动** Pull/Push、实时同步、推送通知。  
- **打开应用即抢锁** 或「持锁期间禁止本地编辑」的强单写模式（一期仅 Push 时互斥）。  
- **实体级 / 表级增量合并**、会话 fork 式冲突保留、CRDT。  
- **服务商三表**（`sksp_secrets`、`llm_provider`、`llm_saved_model`）的跨端同步。  
- **CLI** 云同步子命令（可后续单独立项）。  
- **iOS** Mobile 验收。  
- **WebDAV 专用协议**（若厂商仅提供 WebDAV 而无 S3 API，一期不支持，可后续扩展）。  
- 云端快照 **加密**（用户桶侧服务端加密除外）、多版本快照管理 UI（一期仅保留当前 rev 对应快照；历史 rev 是否物理删除由实现决定，不做产品承诺）。  
- 统一 `nm-desktop-ui` / `nm-mobile-ui` 跨端 UI 偏好同步。  
- 本 PRD **不展开** 接口设计、对象路径、条件写入、任务拆分（见后续 SPEC）。

## 核心需求

1. **配置云存储**：用户可保存 S3 兼容存储参数并通过测试连接；配置仅本机保存，不上传 Novel Master 官方服务器。  
2. **手动 Pull**：从云端拉取最新快照并导入本地，保留本机服务商；导入前可恢复本地备份。  
3. **手动 Push**：将本机数据以 `.nmbackup` 语义上传；成功则远端版本号递增并释放锁。  
4. **Push 互斥**：Push 开始时申请租约锁；他人持有效锁时本机 Push 失败并提示，不得静默覆盖。  
5. **先拉后推**：云端版本高于本机已同步版本时，默认禁止 Push，要求先 Pull 或显式确认覆盖云端。  
6. **锁超时自愈**：锁过期后其他设备可 Push；Push 失败时尽量主动释放锁。  
7. **双端一致**：Mobile 与 Desktop 在数据管理区提供等价能力与一致用户文案；与现有本地导入/导出并存。

## 验收标准

### 配置

- **Given** 用户填入有效的 S3 兼容参数  
  **When** 点击「测试连接」  
  **Then** 提示成功，且可保存配置；无效参数提示失败原因，不保存错误配置。

- **Given** 尚未配置云存储  
  **When** 用户点击 Pull 或 Push  
  **Then** 提示先完成配置，不发起同步。

### Pull

- **Given** 云端版本高于本机，且存在有效快照  
  **When** 用户确认 Pull  
  **Then** 本机项目/会话/消息等工作数据与云端一致；本机导入前服务商配置在 Pull 完成后 **仍可调用**；本机「已同步版本号」与云端一致。

- **Given** 云端与本机版本相同  
  **When** 用户 Pull  
  **Then** 提示已是最新或等价语义，不破坏本机数据。

- **Given** Agent 正在运行  
  **When** 用户 Pull  
  **Then** 与导入数据库一致：拒绝或提示稍后再试（行为与现有导入策略一致）。

### Push

- **Given** 本机无未同步改动或已与云端版本对齐  
  **When** 用户 Push 且无人持有效锁  
  **Then** 上传成功，云端版本号 +1，编辑锁释放，本机「已同步版本号」更新。

- **Given** 云端版本高于本机已同步版本  
  **When** 用户 Push  
  **Then** **默认阻断**，提示须先 Pull；仅在用户二次确认「覆盖云端」后才可继续（若产品保留该路径）。

- **Given** 另一台设备正持有未过期编辑锁  
  **When** 本机 Push  
  **Then** Push 失败，提示另一台设备正在同步/编辑，**不**上传快照、**不**递增云端版本号。

- **Given** Agent 正在运行  
  **When** 用户 Push  
  **Then** 拒绝并提示（与导出数据库一致）。

### 锁超时

- **Given** 设备 A Push 过程中异常退出，锁未主动释放  
  **When** 锁过期后设备 B Push  
  **Then** B 可成功 Push，无需在 A 上操作。

### 与本地备份的关系

- **Given** 用户曾使用本地「导出数据库」  
  **When** 启用云同步后 Pull/Push  
  **Then** 本地导出/导入入口仍可用，行为与迭代前一致。

### 双端互通（端到端）

- **Given** Desktop 创建项目与会话并 Push；Mobile 配置同一桶  
  **When** Mobile Pull  
  **Then** Mobile 可见同一项目与会话数据。

- **Given** Mobile 修改会话并 Push；Desktop 已配置同一桶且云端版本落后  
  **When** Desktop 先 Pull 再打开会话  
  **Then** Desktop 可见 Mobile 的修改。

- **Given** 双端均已配置各自服务商  
  **When** 完成上述 Pull/Push 往返  
  **Then** 双端各自服务商与 API Key **仍可用**。

## 约束与依赖

- 依赖 `import-export-navigation-fix` 交付的 **导出排除 / 导入保留服务商三表** 能力；云快照与本地导出语义一致。  
- 依赖各端既有 **数据库导入前备份、Agent 运行守卫、导入后 rebootstrap** 行为。  
- 用户自行承担对象存储费用与凭据安全；应用不提供官方桶。

## 非功能需求（业务/体验）

- Push/Pull 过程展示进度或忙碌状态，允许用户等待，避免重复点击。  
- 错误信息区分：**网络失败**、**凭据错误**、**他人持锁**、**须先 Pull**、**Agent 运行中**，不混为一谈。  
- 首次配置成功后，数据管理页可一眼看出「已连接 / 未连接」与版本关系。

## 风险与待确认项

| 风险 | 缓解 |
|------|------|
| 双端离线各自修改后轮流 Push | 一期靠「先拉后推」与整库替换；后 Push 者覆盖先 Push 者，须在文案中说明 |
| 大体积数据库上传失败 | Push 失败释放锁 + 租约续期；提示重试 |
| 用户误配他人桶 | 测试连接 + 可选路径前缀隔离 |

- **待后续 SPEC 细化**：对象路径命名、租约默认时长、是否保留「强制覆盖云端」二次确认路径。

## 里程碑（可选）

1. **一期（本 PRD）**：手动 Pull/Push + 租约锁 + 版本号 + 双端设置页。  
2. **后续（另立项）**：后台定时 Push、多快照历史、WebDAV 驱动、实体级合并、CLI 支持。
