# agent-resilience-mobile-yaml PRD

## 背景

当前 Agent 运行链路在稳定性与大会话场景下存在多个薄弱点：缺少手动终止、弱网下流式消费存在背压风险、消息读取一次性全量加载、LLM transient error 无统一重试、doom loop 仅单轮检测、以及 mobile 端缺少 Agent/事件配置 YAML 导入导出能力。以上问题会在移动端弱网与长会话中放大，影响可恢复性与可用性。

## 目标（含成功指标）

- 在不做大规模架构重构前提下，优先提升 Agent 运行稳定性与移动端可恢复性。
- 完成一组“可上线验证”的优化闭环：失败可重试、运行可终止、长会话可分页、循环可检测、配置可导入导出。
- 成功指标（阶段性）：
  - 以“先优化验证”为准，本期不设硬性量化阈值。
  - 验收以功能可用、异常路径可处理、弱网/长会话下行为正确为主。

## 用户与场景

- 目标用户：移动端最终用户。
- 典型场景：
  - 弱网或抖动网络下继续/恢复 Agent 运行。
  - 长会话（大量历史消息）下打开会话并继续对话。
  - Agent 运行异常时手动中止，避免资源持续消耗。
  - 在 mobile 端迁移/备份 Agent 与事件配置（YAML 导入导出）。

## 范围

### 包含范围

- Agent 手动终止能力（运行中可取消）。
- 空执行规则优化：
  - 输入框为空时，若最后一条消息是 `user`，允许继续执行（用于网络中断后的恢复）。
  - 不将空字符串作为新消息发送给模型，而是基于当前会话上下文继续请求。
- LLM 请求重试机制：
  - 对 transient error（如 429、5xx、网络抖动）进行重试。
  - 支持指数退避。
  - 重试次数可配置。
- Streaming 背压治理：
  - 为 `onStream` 消费链路提供背压/缓冲控制，避免慢消费导致内存持续膨胀。
- 消息分页能力：
  - 会话消息读取支持分页（如 limit/offset 或 cursor 等等价能力）。
  - 避免长会话一次性全量加载。
- Doom loop 跨轮检测：
  - 能识别跨轮循环模式（如 A->B->A->B）并触发终止/保护策略。
- Mobile UI：
  - Agent 运行中，发送按钮切换为“终止”icon（点击可取消当前运行）。
  - 当输入框为空但最后一条消息是 `user` 时，发送按钮保持可用（不禁用）。
  - Agent YAML 导入/导出。
  - 事件配置 YAML 导入/导出。

### 不包含范围

- 按厂商定制的精细化重试策略（逐厂商错误码深度映射与专属参数调优）。
- 消息存储层的大规模重构或数据迁移。
- 非 mobile 端的大规模 UI 改造。
- 完整可观测平台接入（看板、告警体系等）。

## 核心需求（3-7 条）

1. 系统必须支持在 Agent 运行过程中由用户手动终止，并保证终止后不再继续后续轮次调用。
2. 系统必须支持“恢复型空执行”：当输入为空且最后一条为 `user` 时，可直接继续运行当前会话上下文。
3. 系统必须在 LLM 请求出现 transient error 时执行可配置次数的重试，并采用指数退避策略。
4. 系统必须在流式输出链路中提供背压控制，确保慢消费场景下不会出现无界内存增长。
5. 系统必须提供消息分页读取能力，避免长会话全量加载带来的显著延迟和内存压力。
6. 系统必须具备跨轮 doom loop 检测能力，能识别并阻断跨轮重复循环模式。
7. Mobile 端必须支持 Agent 与事件配置的 YAML 导入/导出，导入失败时提供可理解错误信息；并满足运行态按钮切换与“最后一条为 `user` 时可空执行”的交互规则。

## 验收标准

- [ ] 用户在 Agent 运行中触发“终止”后，当前 run 在可接受时间内停止，且不会继续发起新的模型请求。
- [ ] Mobile UI 在 Agent 运行中将发送按钮显示为“终止”icon，点击后可触发手动终止。
- [ ] 当输入框为空且会话最后一条是 `user` 时，触发运行不会追加空消息；系统能基于现有会话继续请求模型。
- [ ] 当输入框为空但会话最后一条是 `user` 时，发送按钮不被禁用，可直接触发“恢复型空执行”。
- [ ] 当出现 429、5xx 或网络抖动时，系统按配置执行重试与指数退避；超过重试上限后返回明确失败结果。
- [ ] 重试次数配置可被读取并生效，配置变更后无需改代码即可调整行为。
- [ ] 在慢速流式消费场景下，系统能通过背压/缓冲策略保持稳定，不出现明显内存失控或进程异常。
- [ ] 长会话读取可按分页获取，首轮加载不再依赖全量 `listBySession()` 结果。
- [ ] 出现跨轮工具调用循环（如 A->B->A->B）时，系统可识别并中断，给出可追踪原因。
- [ ] Mobile 端可完成 Agent YAML 导入/导出与事件配置 YAML 导入/导出；无效 YAML 会被拒绝并提示错误原因。
