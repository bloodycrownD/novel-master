# annotate-custom-highlight-soft-range PRD

## 背景

聊天会话工作区里，用户可以在文件预览中划词加批注，预览用下划线标出原文，Composer 出现「批注:路径」chip，发送时把批注交给模型。现在高亮靠「原文在整篇预览里尽力搜索」，再在 DOM 里包一层或多层 `<mark>`。跨加粗、链接、表格单元格时，经常对不齐或看起来像没标上；文本预览里多行划词也容易对不上——不是因为文本模式另搞一套，而是同一套「全文串匹配 + 改 DOM」在换行和复杂结构上本来就脆。

此前 `annotate-cross-node-highlight` 用扁平文本索引和多段 mark，目标仍是跨节点能看见下划线，但合同里写明本期不做 CSS Custom Highlight，也不把 offset 落进库。`annotate-workplace-ux-fix` 又为表格跨格去掉 needle 里的换行，换来表内更好拼，却加重了文本/`<pre>` 多行的命中问题。

本迭代要换一条更稳的产品路径：**真正把跨节点批注做成可用**（必要时替换现有 mark 主路径，而不是为了回滚而回滚）；展示优先用浏览器的自定义高亮能力；批注附带**宽松行列窗口**，既帮预览缩小搜索范围，也随附件给模型/search 用。原文匹配仍然保留。范围故意松一点，这样前面改了几行字，批注也不至于整条「蒸发」。

## 目标（含成功指标）

| 目标 | 成功指标 |
|------|----------|
| 跨节点可见高亮 | 同一预览内，跨加粗/链接等行内节点的连续划词，添加批注后能看到连续（或多段观感连续）高亮 |
| 文本与 Markdown 同体验 | 会话工作区预览下，文本 Tab 与 Markdown Tab（及 Desktop 等价路径）都能划词、高亮、点开 |
| 宽松行列辅助定位 | 新建批注带有起止行列窗口；预览匹配优先在窗口内找原文；发给模型的批注附件里也能看到该窗口 |
| 前文小改不丢条目 | 窗口内或附近原文仍在时，高亮应尽量还在；即使暂时对不上下划线，chip 与草稿条目仍在（不低于现网底线） |
| 不支持新 API 的环境可用 | 无 Custom Highlight 时自动回退到现有（或简化的）mark 绘制，核心划词/chip/发送不断 |

## 用户与场景

| 用户 | 场景 |
|------|------|
| 写作者 | 在会话文件预览里划过加粗标题或表格里的句子，加批注后立刻能看见标记并点开修改 |
| 文本稿作者 | 切换到「文本」预览划多行注释，行为与 Markdown 预览一致，不会「只有 chip 没有线」 |
| 依赖模型定位的作者 | 发送后模型能根据宽松行列 + 原文更快在文件里找到批注位置，而不是整文件盲搜 |
| 旧设备用户 | WebView 较旧时仍能划词批注；高亮可能走 mark 回退，但流程完整 |

## 范围

### 包含范围

1. **预览高亮主路径**改为可跨节点的自定义高亮（Custom Highlight）；不支持时回退 mark（或多段 mark），双端（Desktop / Mobile 会话预览）行为合同一致。  
2. **文本预览与 Markdown 预览**共用同一套「采集 → 存储 → 绘制 → 点开」合同（渲染器可以不同，用户可感知行为须一致）。  
3. **批注数据**在保留 `path` / `originalText` / `userAnnotation` 的前提下，增加**可选的宽松起止行列**（可含列；窗口可大于真实选区）；写入草稿，并进入发送给模型的批注附件。  
4. **预览匹配**：有行列时先在窗口内（可再略放大）用原文匹配；没有行列的旧批注仍按现网全文原文匹配。  
5. **点高亮打开批注**的能力保留（实现方式可随 Custom Highlight 调整，但用户仍能从高亮区域打开对应条目）。  
6. 明确本迭代是**替换/演进**跨节点高亮手段，**不是**发版「退回仅单 Text 初版」；若实现过程中去掉旧 flat-mark 主路径，须用新路径达到同等或更好的跨节点可用性。

### 不包含范围

- 恢复消息气泡正文划词批注。  
- 项目/全局工作区预览批注（门闩仍限于聊天会话工作区）。  
- 把批注改成写回 Markdown 语法（如 `==高亮==`）；高亮仍是预览层效果。  
- 精确到字符、改一字就失效的刚性锚点；也不做完整 OT/CRDT。  
- 强制统一 Mobile markdown-it 与 Desktop remark 的渲染引擎。  
- 云同步冲突策略、CLI 批注子命令。  
- 修改 workplace 常驻配置、FileEditor 文件名居中等与本主题无关的项。

## 核心需求（3-7 条）

1. **跨节点高亮可用**：用户在预览中划选跨行内节点的连续可见文本并保存批注后，应能看到对应高亮。  
2. **Custom Highlight 优先，mark 可回退**：宿主支持时用自定义高亮绘制；不支持时回退 mark，划词、chip、发送仍可用。  
3. **文本与 Markdown 同合同**：两种预览模式下，划词、高亮、点开、chip 行为一致。  
4. **宽松行列双用途**：新建批注采集宽松起止行列；既用于预览缩小匹配范围，也随批注附件交给模型/search。  
5. **原文仍在**：`originalText` 继续作为匹配与展示依据；行列是辅助窗口，允许比真实选区更大。  
6. **存量兼容**：旧批注没有行列字段时，行为不低于现网（全文原文尽力匹配；chip 不因匹配失败消失）。  
7. **可点开**：用户点击高亮区域仍能打开对应批注（多段/多 Range 时任一段可打开同一批）。

## 验收标准

### 跨节点与双预览

- **Given** 会话工作区预览打开含加粗/链接的 Markdown  
  **When** 用户划选跨这些行内样式的连续文本并添加批注  
  **Then** 预览出现对应高亮，且可点击打开该批注。

- **Given** 同一文件切换到文本预览（或 Desktop 非 md 预览）  
  **When** 用户划选多行文本并添加批注  
  **Then** 高亮可见（在支持的宿主上）；chip 出现；行为与 Markdown 预览合同一致。

### 宽松行列

- **Given** 用户新建一条批注  
  **When** 查看草稿或发送后的批注附件内容  
  **Then** 含有可选的起止行列信息，且窗口覆盖（可大于）选区所在区域。

- **Given** 批注带有行列窗口，且原文仍出现在该窗口附近  
  **When** 刷新预览高亮  
  **Then** 优先在窗口内匹配原文并显示高亮，而不是一上来整篇误匹配无关处。

- **Given** 旧消息/旧草稿没有行列字段  
  **When** 打开预览或 Undo 恢复  
  **Then** 仍可按原文尽力匹配；解析不失败；chip 逻辑不低于现网。

### 回退与底线

- **Given** 宿主不支持 Custom Highlight  
  **When** 用户完成划词批注  
  **Then** 仍有可见高亮（mark 回退）或至少 chip/条目保留且可编辑；发送路径可用。

- **Given** 暂时无法匹配出高亮  
  **When** 用户查看 Composer  
  **Then** 「批注:路径」chip 与草稿条目仍在（与现网尽力匹配底线一致）。

## 风险与待确认项

- **坐标系**：行列以磁盘源文件为准（推荐），还是以预览 DOM 可见行为准；Markdown 渲染后行号与源文件可能不一致，PRD 验收以「对用户足够、窗口偏松」为准，精确映射留给实现说明。  
- **松弛幅度**：默认放大几行/几列尚未钉死；实现可选固定 ±N 行，验收只要求「明显宽于选区且仍有助于定位」。  
- **点击命中**：Custom Highlight 不产生 `<mark>` 节点时，点选实现须另定，但不能取消「点高亮可打开」。  
- **WebView 碎片**：Android System WebView 版本不一，必须以运行时探测 + mark 回退为准，不能假定全员支持。  
- **与表连续合同**：跨格/整行「观感连续」仍希望保留；若与 Custom Highlight 或行列映射冲突，优先保证跨节点与文本预览可用，表连续可作为同迭代尽力项或 follow-up。  
- **旧文档 D6/T5**：本迭代正式允许 Custom Highlight 作为预览主路径，并允许 optional 行列进入草稿/附件；以此覆盖此前「本期不做 / 不改落库」的边界。
