# Mobile Bugfix PRD

> **平台**：Android Mobile（本迭代验收范围）  
> **类型**：体验修复 / 配置简化  
> **关联迭代**：`mobile-vfs-markdown-webview`、`provider-model`、`mobile-app`

## 背景

Mobile 端存在三处可复现的体验问题：

1. **文件预览**：非 Markdown 文件（如 `.txt`）在预览模式下内容超出屏幕时 **无法滑动**，长文无法阅读；Markdown 文件预览正常。
2. **模型展示**：用户在「我的 → 当前模型」切换工作区模型后，已打开的聊天页右上角模型标识 **不更新**，与 `PersistentState` 中 `currentModelId` 的实际值不一致，造成「改了但没生效」的错觉。
3. **服务商配置**：「服务商展示名称」（`displayName`）字段在 UI 中可编辑，但 **不影响任何展示**（列表/详情/聊天页均不读取该值），属于无效配置项，增加认知负担。

## 目标（含成功指标）

| 目标 | 成功指标 |
|------|----------|
| 文件预览可滚动 | 任意文件类型在预览模式下，内容超出视口时可 **流畅纵向滑动** 阅读全文 |
| 模型展示同步 | 切换「当前模型」后，聊天页模型标识 **立即** 与 `currentModelId` 一致 |
| 配置项精简 | 移除无效的「服务商展示名称」字段；相关 UI 统一以 **服务商 ID** 展示 |

**整体成功指标**：上述三问题在 Android 手工验收用例中 **全部通过**，无回归。

## 用户与场景

| 用户 | 场景 |
|------|------|
| 工作区/会话文件阅读用户 | 在文件编辑器预览 `.txt`、`.json` 等非 `.md` 长文件，需上下滑动阅读 |
| Markdown 文件用户 | 打开 `.md` 文件，默认 Markdown 预览；可切换「文本」Tab 查看源码 |
| 模型切换用户 | 在「我的」页切换当前模型后，回到聊天页确认当前生效模型 |
| 服务商配置用户 | 添加/编辑服务商时，不再看到无意义的「展示名称」字段 |

## 范围

### 包含范围

1. **文件预览滚动修复**
   - 预览模式对所有文件类型均支持 **Markdown / 文本** 两种渲染（与现有 SegmentedControl 行为一致）。
   - **`.md` / `.markdown` 文件**：进入预览时 **默认 Markdown Tab**；用户可切至「文本」查看源码。
   - **其他文件**：进入预览时 **默认文本 Tab**（纯文本/monospace 展示）；用户可切至「Markdown」尝试渲染（若内容含 Markdown 语法）。
   - 无论哪种 Tab、何种扩展名，预览区域在 `previewFill` 布局下 **必须可滚动**。

2. **聊天页模型标识同步**
   - 模型标识的数据源为 **单一真相**：`runtime.state.getCurrentModelId()`（经 agent 模型 pin 解析后的 `applicationModelId`）。
   - 用户在「我的 → 当前模型」完成切换后，**无需离开/重进会话**，当前聊天页右上角模型标识 **立即更新**。
   - Agent 有独立模型 pin 时，仍显示 agent 解析后的模型（现有逻辑不变）。

3. **移除服务商展示名称**
   - 从服务商 **创建/编辑表单** 移除「显示名称」输入项。
   - 服务商 **列表、详情页标题、导航栏** 等原使用 `displayName` 的位置，统一改为展示 **服务商 ID**（如 `zhipu`、`openai`）。
   - 已有数据库中的 `displayName` 字段可保留（向后兼容），但 **不再暴露给用户编辑**，也不再参与 UI 展示。

### 不包含范围

- iOS 验收（可后续跟进）。
- Desktop / CLI 端同类问题。
- 模型 **展示格式** 改造（如改为短标签、自定义 alias）；本迭代仅修复 **不同步** 问题。
- 服务商 **模型** 的 `displayName`（保存模型时的可选别名）——与本迭代「服务商级 displayName」无关。
- 文件预览引擎切换（WebView vs RN 渲染）策略变更。
- 数据库 migration 删除 `displayName` 列（非必须，可留待清理迭代）。

## 核心需求

1. **非 md 文件预览可滚动**：修复 `FileMarkdownPreview` / `FileEditorScreen` 在非 Markdown 路径下 `previewFill=true` 时缺少 `ScrollView` 的问题，使长文本可滑动。
2. **预览 Tab 默认值按扩展名区分**：`.md` 默认 Markdown；其他文件默认文本；两种 Tab 对所有文件可用。
3. **聊天页订阅模型变更**：「我的」页切换 `currentModelId` 后，聊天 Tab 已打开的会话页 **即时刷新** 模型标识（无需重进会话）。
4. **模型展示仍走单一数据源**：展示值来自 `getCurrentModelId()` + agent pin 解析，不引入第二份缓存。
5. **移除服务商展示名称 UI**：创建/编辑表单去除该字段；列表与详情标题改用 **服务商 ID**。
6. **无功能回归**：Markdown 预览（含 Front Matter、WebView 引擎）、文件编辑/保存、模型选择与 Agent pin 逻辑保持现有行为。

## 验收标准

### 1. 文件预览滚动

- **Given** 会话或项目中存在内容超过一屏的 `.txt` 文件  
  **When** 打开文件编辑器并处于 **预览模式**（默认）  
  **Then** 用户可在预览区域内 **上下滑动** 阅读全文，无内容被截断且无法滚动的情况。

- **Given** 存在 `.md` 文件  
  **When** 进入预览模式  
  **Then** 默认选中 **Markdown** Tab，渲染正常；切换至「文本」Tab 可查看源码且可滚动。

- **Given** 存在非 `.md` 文件（如 `.txt`）  
  **When** 进入预览模式  
  **Then** 默认选中 **文本** Tab；若用户切换至 Markdown Tab，预览区域仍可滚动。

- **Given** 用户在预览与编辑模式间切换  
  **When** 再次进入预览  
  **Then** 滚动与 Tab 选择行为符合上述规则，编辑/保存功能正常。

### 2. 模型标识同步

- **Given** 用户已打开某会话的聊天页，当前显示模型为 `A`  
  **When** 切换至「我的 → 当前模型」并选择模型 `B`，再返回聊天 Tab（同一会话仍打开）  
  **Then** 聊天页右上角模型标识 **立即变为 `B`**（或与 `B` 对应的展示标签），无需重进会话。

- **Given** 当前 Agent 配置了独立模型 pin（非工作区模型）  
  **When** 用户切换工作区当前模型  
  **Then** 聊天页仍显示 **Agent pin 的模型**（现有优先级不变）。

- **Given** 用户切换模型后发送新消息  
  **When** 消息发出  
  **Then** 实际调用模型与页头展示一致（沿用 `currentModelId` 解析链路）。

### 3. 移除服务商展示名称

- **Given** 用户进入「添加服务商」或「编辑服务商」  
  **When** 查看表单  
  **Then** **不存在**「显示名称 / 服务商展示名称」输入项。

- **Given** 已存在的服务商（无论是否曾填写 displayName）  
  **When** 在服务商列表或详情页查看  
  **Then** 标题/名称展示为 **服务商 ID**（如 `zhipu`），而非 displayName。

- **Given** 用户完成服务商创建或编辑  
  **When** 保存  
  **Then** 流程正常，无因移除字段导致的崩溃或校验错误。

### 4. 回归

- Markdown 文件 Front Matter 卡片、WebView 预览引擎切换仍正常。
- 「我的 → 当前模型」选择器、Agent 模型 pin 配置仍正常。
- 服务商 API Key 保存、模型列表获取仍正常。
