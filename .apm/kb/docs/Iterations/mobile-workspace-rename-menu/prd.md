# Mobile 工作区命名统一与会话菜单精简 PRD

> **平台**：Android + iOS（`apps/mobile`）  
> **性质**：文案与交互重组 + 批量多选圈选框尺寸微调，**不改变** VFS 路径、路由名、Core API 或批量操作语义。  
> **关联截图**：会话操作抽屉（汉堡菜单）、聊天/工作区 Tab、消息批量多选态。

## 背景

Mobile 中三级 VFS 浏览入口长期混用「**模板**」与「**工作区**」：

| 层级 | 现用户可见文案 | 问题 |
|------|----------------|------|
| 全局 | 全局模板 | 「模板」易与「可复制骨架」混淆，实际是会话外共享文件区 |
| 项目 | 项目模板 | 同上 |
| 会话 | 会话工作区 | 与会话级「聊天」Tab 并列时，命名层级不直观 |

此外，会话操作抽屉中 **批量删除 / 批量隐藏 / 批量取消隐藏** 占三行，菜单过长；进入批量模式后每次仅暴露一种操作，需从菜单反复进入。

消息批量多选时，聊天气泡左侧 **圈选框视觉偏大**（WebView transcript 默认路径尤为明显），挤占横向空间、与气泡比例失调。

产品决策：统一三级为「**工作区**」心智；批量相关能力合并为「**批量操作**」，批量顶栏同时提供 **删除、隐藏、恢复** 三个按钮；圈选框 **缩小视觉尺寸**，整行仍可点选。

## 目标（含成功指标）

| 目标 | 成功指标 |
|------|----------|
| 命名一致 | 用户在 Profile、项目抽屉、会话对话页三处，均能识别「全局 / 项目 / 聊天」三级 **工作区**，不再出现「模板」字样（用户可见处） |
| 菜单精简 | 会话操作抽屉操作项由 8 行减至 **6 行**（合并 3 项批量为 1 项） |
| 批量效率 | 进入批量模式后，**无需返回菜单**即可在删除 / 隐藏 / 恢复间切换执行 |
| 语义清晰 | 「聊天重命名」「查看提示词」「压缩上下文」与功能一一对应，用户无歧义 |
| 多选圈选框比例 | 批量模式下圈选框 **视觉直径/边长明显小于现网**（主观对照截图），不遮挡气泡首行文字；点选区域仍 ≥ 44pt 等效（靠行内 padding / hitSlop，非放大控件本身） |

## 用户与场景

| 用户 | 场景 |
|------|------|
| 日常写作用户 | 在「聊天」与各级工作区间切换，理解三栏文件归属 |
| 会话整理用户 | 通过会话菜单批量删除/隐藏/恢复历史消息 |
| 调试用户 | 查看提示词、手动触发压缩上下文 |

## 范围

### 包含

**A. 工作区命名（用户可见文案）**

| 现文案 | 新文案 | 典型触点 |
|--------|--------|----------|
| 全局模板 | **全局工作区** | Profile 入口、`GlobalTemplate` 导航标题、`GlobalTemplateScreen` 提示 |
| 项目模板 | **项目工作区** | 项目抽屉 SegmentedControl、`AppHeader` 标题 |
| 会话工作区 | **聊天工作区** | 会话对话页 SegmentedControl、不可用提示、工具卡片「点击查看 · …」、`RealPromptScreen` 说明文案等 |

> 注：实现时「模板」「模版」等异体字一并替换；`从上级同步` 等确认框中的「模板」改为对应「工作区」表述。

**B. 会话操作抽屉（`SessionActionsDrawer`）**

| 现文案 | 新文案 |
|--------|--------|
| 重命名 | **聊天重命名** |
| 真实提示词 | **查看提示词** |
| 压缩会话 | **压缩上下文** |
| 批量删除消息 / 批量隐藏消息 / 批量取消隐藏 | 合并为一项：**批量操作** |

保留不变（本期）：切换模型、切换 agent、当前 agent/模型信息区。

**C. 批量消息 UI**

- 从菜单点「批量操作」进入消息多选模式（行为与现「批量删除」入口一致：Agent 运行中禁止进入）。
- 批量顶栏（现 `ManageHeader` 或等价组件）在已选消息时 **同时展示三个操作按钮**：
  - **删除**（危险色，需二次确认）
  - **隐藏**（需二次确认）
  - **恢复**（即现「取消隐藏」，需二次确认；无选中时按钮置灰）
- 取消批量、已选计数展示保持现网行为。

**D. 消息多选圈选框尺寸（批量模式）**

- **主路径（WebView transcript）**：缩小 `transcript-html.ts` 中 `.batch-check` 的视觉尺寸（现约 28×28px 圆角圈）；同步收紧 `.batch-row` 间距，使圈选框与气泡垂直对齐更紧凑。
- **回滚路径（legacy RN `MessageList`）**：`BatchCheckbox` 与 `batchCheckboxCol` 列宽与主路径 **视觉一致**（现 RN 方框 22×22、列宽 36，按需同比例缩小）。
- **交互不变**：点整行或点圈选框均可切换选中；选中态（描边/填充/✓）语义不变。
- **可访问性**：缩小的是 **绘制尺寸**，点击热区通过 `batch-row` 行高或 `hitSlop` 补足，避免误触困难。

**E. 关联文案联动**

- 导航配置：`RealPrompt` 标题 → **查看提示词**
- 压缩确认 `Alert` 标题：「压缩会话」→ **压缩上下文**（正文与 Toast 可沿用或微调为「上下文」措辞）
- `TemplatePullButton` 覆盖确认文案中的「模板」→ 对应「工作区」
- 其他 grep 到的用户可见「会话工作区」「全局/项目模板」「真实提示词」「压缩会话」字符串

### 不包含

- `examples/desktop`、`examples/mobile` 原型文档同步（可选，非验收）
- 历史迭代 PRD/SPEC 文档中的旧术语回改
- 路由名（`GlobalTemplate` 等）、TypeScript 标识符、KKV key 重命名
- 新增第四种批量操作或消息菜单项
- 项目/Agent/VFS 文件等其他场景的「重命名」文案（仍为「重命名项目」等）

## 核心需求

1. **三级工作区命名统一**：全局 / 项目 / 聊天工作区，全应用用户可见处一致。
2. **会话菜单四项重命名**：聊天重命名、查看提示词、压缩上下文、批量操作（合并入口）。
3. **批量三按钮并列**：删除、隐藏、恢复同屏可选，各自确认后执行，执行完毕退出批量模式。
4. **联动文案不遗漏**：Tab、Header、抽屉、Alert、Toast、空态/提示、工具卡片 hint 一并更新。
5. **行为不退化**：现有批量/压缩/重命名/查看提示词功能逻辑不变，仅改入口与标签。
6. **圈选框视觉缩小**：WebView 与 legacy RN 两条 transcript 路径的多选控件尺寸协调缩小，不牺牲可点性。

## 验收标准

### 命名（T1–T6）

- **T1** Profile → 入口显示 **全局工作区**；进入后导航标题为 **全局工作区**。
- **T2** 项目抽屉 → SegmentedControl 第二项为 **项目工作区**（非「项目模板」）；对应 Header 标题一致。
- **T3** 会话对话页 → SegmentedControl 第二项为 **聊天工作区**（非「会话工作区」）。
- **T4** 工具调用卡片 / WebView hint 中「点击查看 · …」后缀为 **聊天工作区**。
- **T5** 「从上级同步」类确认框文案使用 **全局工作区 / 项目工作区 / 聊天工作区**，无「模板」字样。
- **T6** 全应用用户可见字符串检索（QA 清单）：无遗漏的「全局模板」「项目模板」「会话工作区」（本迭代应改范围）。

### 会话菜单（T7–T10）

- **T7** 打开会话操作抽屉：可见 **聊天重命名、切换模型、切换 agent、查看提示词、压缩上下文、批量操作** 共 6 项操作（不含信息区）；**无** 原三条独立批量项。
- **T8** 点 **聊天重命名** → 行为与现「重命名」一致（会话标题编辑）。
- **T9** 点 **查看提示词** → 进入全屏页，标题为 **查看提示词**。
- **T10** 点 **压缩上下文** → 确认框标题含 **压缩上下文**；确认后压缩行为与现网一致。

### 批量操作（T11–T15）

- **T11** 点 **批量操作** → 进入消息多选；顶栏显示取消、已选计数、**删除 / 隐藏 / 恢复** 三个按钮。
- **T12** Agent 运行中点 **批量操作** → Toast 提示与现网一致（禁止进入）。
- **T13** 选中 ≥1 条消息后点 **删除** → 二次确认 → 删除选中消息并退出批量模式。
- **T14** 选中 ≥1 条消息后点 **隐藏** → 二次确认 → 隐藏选中消息并退出批量模式。
- **T15** 选中 ≥1 条消息后点 **恢复** → 二次确认 → 取消隐藏选中消息并退出批量模式；未选中时三按钮均不可点或置灰。

### 多选圈选框（T16–T17）

- **T16** WebView transcript 批量模式：圈选框视觉尺寸 **小于** 迭代前（对照用户截图），与 15px 字号气泡比例协调；选中/未选中态清晰可辨。
- **T17** legacy RN `MessageList` 批量模式（`chatTranscriptEngine=legacy-rn`）：圈选框视觉尺寸与 WebView 路径 **无明显两套风格**；点选与整行切换仍正常。

### 回归（T18）

- **T18** 切换模型、切换 agent、编辑↔聊天工作区 Tab、WebView/RN transcript 批量勾选均正常，无布局错位或菜单项失效。

## 约束与依赖

- 仅 `apps/mobile` 工程；无 Core / 数据库变更。
- 依赖现有 `SessionActionsDrawer`、`ManageHeader`、`messageBatch` 状态机。
- 「恢复」=user-visible 标签；底层仍调用 `messages.show`（取消隐藏）。

## 风险与待确认项

| 风险 | 缓解 |
|------|------|
| 遗漏散落文案 | 实现前 `rg` 全量扫描；T6 作 QA gate |
| `ManageHeader` 仅支持单主按钮 | SPEC 阶段扩展组件或会话批量专用顶栏 |
| 圈选框缩小后难点按 | 热区放在整行 `batch-row`，非仅缩小后的圆心 |
| 用户习惯「模板」旧称 | 本期一次性切换，无灰度 |
