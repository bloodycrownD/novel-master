# 移动端原型 — 会话操作抽屉与会话日志 PRD

## 背景

`examples/mobile` 为工作区与协作的 **UI 壳原型**（可直接打开 `index.html` 验收交互）。当前顶栏右侧 ☰ 统一打开 **项目列表** 左侧抽屉；进入会话聊天后，输入区上方仍有 **真实提示词 / 工具日志 / 检查点** 三个 Chip，分别压栈进入三个独立全屏页。

该布局在「已进入某会话」时仍暴露项目抽屉，且调试类能力分散在 Chip 上，与「会话列表 vs 会话内」的信息层级不一致。需要将 **项目切换** 与 **当前会话工具** 分离，并合并日志与检查点的浏览入口。

## 目标（含成功指标）

| 目标 | 成功指标 |
|------|----------|
| 顶栏入口按上下文分流 | 会话列表子视图下，☰ 打开 **项目抽屉**；进入某会话后（含「聊天 / 会话工作区」上 Tab），☰ 打开 **会话操作** 左侧抽屉 |
| 会话能力集中入口 | **会话操作** 抽屉仅 **2 项**：真实提示词、会话日志 |
| 移除冗余 Chip | 聊天输入区 **不再展示** 真实提示词 / 工具日志 / 检查点 Chip |
| 日志与检查点合一 | 仅保留 **会话日志** 全屏页；工具执行与检查点创建在 **统一时间线** 中按时间倒序展示 |
| 能力不降级 | 原工具日志页中的 **关联检查点、从记录发起回滚** 等能力在会话日志时间线中仍可完成（不可用检查点时有明确提示） |

**量化验收（原型阶段）：**

- 在原型中可完成「列表 → 进会话 → 打开会话抽屉 → 进入两个二级页 → 返回」全流程，且无独立「工具日志」「检查点」菜单入口。
- `feature-inventory.md` 或等价设计说明中与 §3.3 Chip、§6 分栏表述 **与 PRD 一致**（可在实现迭代中同步文档）。

## 用户与场景

| 用户 | 场景 |
|------|------|
| 产品 / UI 设计 | 在原型上评审对话 Tab 的信息架构与历史/回滚路径 |
| 开发者（后续接真实 API） | 按统一入口实现真实提示词预览与会话级时间线 |
| 重度写作用户 | 在单会话内查看发给模型的真实提示词，并在一条时间线里追溯工具执行与检查点、必要时回滚 |

## 范围

### 包含范围

1. **顶栏 ☰ 上下文切换**
   - **会话列表子视图**（含「会话 / 项目模板」上 Tab）：☰ → 左侧 **项目抽屉**（项目列表、新建项目、切换项目），行为与现网原型一致。
   - **会话聊天子视图**（已进入某会话，含「聊天 / 会话工作区」上 Tab）：☰ → 左侧 **会话操作** 抽屉（非项目列表）。

2. **会话操作抽屉**
   - 抽屉标题（header）：**会话操作**。
   - 菜单项固定为两项：**真实提示词**、**会话日志**。
   - 点击任一项：关闭抽屉，**压栈进入全屏二级页**（顶栏显示返回，底栏隐藏），与现 Chip 导航模式一致。

3. **移除 Chip 与旧入口**
   - 删除聊天输入区 `context-chips` 中的三个 Chip 及对应交互。
   - **完全移除** 独立全屏页「工具日志」「检查点」及其顶栏/路由入口；仅保留「真实提示词」「会话日志」两个全屏页。

4. **会话日志（合并页）**
   - 全屏页顶栏标题固定为 **会话日志**（UI 中不再出现「工具日志」「检查点」作为页面标题）。
   - 单页 **统一时间线**：工具执行记录、检查点创建（及可区分的其它会话事件类型）按 **时间倒序** 同列展示；条目需能区分类型（文案/图标/标签均可）。
   - 保留业务上必要操作：如从工具执行条目查看摘要、跳转或关联检查点、对仍有效的检查点 **发起回滚**（需确认）；检查点已被 FIFO 淘汰时 **禁用或说明原因**。
   - 页顶或合适位置可保留检查点数量上限等 **用户可理解的提示**（与设置中的保留条数概念一致）。

5. **真实提示词**
   - 全屏页顶栏标题：**真实提示词**。
   - 继续以全屏只读预览呈现；入口仅来自 **会话操作** 抽屉（及实现阶段允许的等价深链，若有）。

6. **原型与文档**
   - 变更落在 `examples/mobile`（`index.html` / `app.js` / `styles.css` 等）。
   - 同步更新 `examples/mobile/docs/feature-inventory.md` 中 §3.3、§6 等与入口相关的用户可见描述（实现任务的一部分）。

### 不包含范围

- 真实后端 API、持久化、检查点引擎实现（仍为原型/mock）。
- 项目抽屉 UI 的重设计（除与顶栏按钮显隐/文案相关的必要调整）。
- 「工作树」独立 Chip 或新入口（工作树仍通过 **会话工作区** Tab 与文件列表规则完成；本需求不新增工作树 Chip）。
- Agent / 我的 / 服务商等其它 Tab 的导航改造。
- 平板分栏、桌面布局。

## 核心需求

1. **上下文感知顶栏 ☰**：在 `chatSubview === sessions` 时打开项目抽屉；在 `chatSubview === conversation` 时打开 **会话操作** 抽屉（标题「会话操作」）。
2. **会话操作抽屉仅两项菜单**：真实提示词、会话日志；选择后全屏压栈导航。
3. **移除聊天区三个 Chip**，避免与抽屉重复入口。
4. **下线工具日志、检查点两个独立全屏页**，路由与 `pageConfig` 仅保留 `realPrompt` 与 `sessionLog`（命名以实现为准）。
5. **会话日志统一时间线**：合并原工具日志列表与检查点列表的浏览与回滚路径；时间线项类型可辨识。
6. **项目切换路径仍可达**：会话列表页通过项目抽屉切换项目；不因会话抽屉而移除项目能力。

## 验收标准

### 顶栏与抽屉

- **Given** 用户位于对话 Tab 的会话列表（未进入任何会话）  
  **When** 点击顶栏右侧 ☰  
  **Then** 从左侧滑出 **项目抽屉**，展示项目列表；**不**出现「真实提示词 / 会话日志」菜单。

- **Given** 用户已进入某一会话（聊天或会话工作区 Tab 均可）  
  **When** 点击顶栏右侧 ☰  
  **Then** 从左侧滑出 **会话操作** 抽屉，标题为「会话操作」，且仅包含 **真实提示词**、**会话日志** 两项；**不**展示项目列表。

- **Given** 会话操作抽屉已打开  
  **When** 用户点击遮罩或完成选择菜单项  
  **Then** 抽屉关闭。

### 导航与入口清理

- **Given** 用户在会话聊天界面  
  **Then** 输入区上方 **不存在** 真实提示词、工具日志、检查点 Chip。

- **Given** 用户在会话操作抽屉  
  **When** 点击「真实提示词」  
  **Then** 进入全屏页，顶栏标题为「真实提示词」，有返回，底栏隐藏；返回后会话聊天状态保持。

- **Given** 用户在会话操作抽屉  
  **When** 点击「会话日志」  
  **Then** 进入全屏页，顶栏标题为「会话日志」；**不出现**「工具日志」或「检查点」作为页面标题。

- **Given** 用户尝试通过 UI 导航  
  **Then** **无法** 进入标题为「工具日志」或「检查点」的独立全屏页（无菜单、无 Chip、无遗留 Tab）。

### 会话日志时间线

- **Given** 用户打开会话日志页  
  **Then** 可见 **单一列表** 按时间倒序排列，且同时包含至少两种可区分类型（如工具执行成功/失败、检查点创建）；**无** 页内「执行记录 | 检查点」双 Tab。

- **Given** 时间线中存在仍有效的检查点关联条目  
  **When** 用户从该条目发起回滚并确认  
  **Then** 出现成功或失败的明确反馈（Toast 或页内文案）；回滚进行中重复操作有禁用或提示。

- **Given** 时间线条目引用的检查点已被淘汰  
  **Then** 回滚入口不可用或展示不可回滚原因（与 feature-inventory §12 边界一致）。

### 项目能力

- **Given** 用户在会话列表切换项目  
  **When** 通过项目抽屉选择另一项目  
  **Then** 当前项目上下文更新，并回到或保持合理的会话列表状态（与现原型「切换项目」行为一致，不因本需求而丢失）。
