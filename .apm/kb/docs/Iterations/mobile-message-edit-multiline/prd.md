# 聊天消息编辑输入框优化 PRD

## 背景

Mobile 聊天中通过长按菜单进入「编辑消息」时，当前使用通用单行 `TextPromptModal`：输入区高度固定、仅一行可见，长消息难以浏览与修改。  
同时，在编辑框内按回车会触发 `returnKeyType="done"` / `onSubmitEditing`，键盘收起或表现为「退出输入」，与底部 **消息输入框**（`multiline`、回车换行、仅点发送提交）的体验不一致，易造成误操作与挫败感。

## 目标（含成功指标）

1. **可读性**：编辑弹窗内能同时看到多行正文（随内容增高，超长可滚动），无需挤在一行里横向扫视。
2. **输入习惯一致**：在「编辑消息」内按回车 **插入换行**，不默认收起键盘、不等同于保存；保存仅通过「保存」按钮（及现有校验逻辑）。
3. **范围收敛**：仅改聊天「编辑消息」路径，不重命名/项目名等其他 `TextPromptModal` 场景。

**成功指标（可观察）**

- 含 ≥10 行文本的历史消息，打开编辑后 **无需缩放或外溢** 即可浏览大部分内容（输入区最小高度 ≥ 底部输入框常态高度）。
- 用户测试：连续按回车 3 次，文本中出现 3 个换行且 **键盘保持打开**（Android 真机）。
- 保存仍走现有 `message-edit` 流程，无回归（隐藏/回滚/Fork 等不受影响）。

## 用户与场景

| 用户 | 场景 |
|------|------|
| 日常聊天用户 | 纠正较长用户消息、补充多段说明 |
| 创作/续写用户 | 编辑含大纲、列表、多段落的 prompt，需在弹窗内看清全文再改 |

## 范围

### 包含范围

- **平台**：Android Mobile（与当前聊天验收面一致）。
- **入口**：聊天 Tab → 长按消息 →「编辑」→「编辑消息」弹窗。
- **交互**：
  - 多行输入（`multiline`），最小高度与底部 `ChatComposer` 输入区视觉接近（可参考其 padding、字号）。
  - 内容增多时输入区增高，达到上限后在输入区内纵向滚动（避免弹窗撑出屏幕）。
  - 回车：换行；**不**因回车触发保存或关闭键盘（移除/避免 `returnKeyType="done"` + `onSubmitEditing` 保存行为）。
  - 保存：仅「保存」；空白/仅空白仍不可保存（与现逻辑一致，是否 trim 首尾空白保持现状）。
- **实现边界（产品层）**：可为消息编辑单独组件或专用 props，**不要求**改动会话重命名、agent 命名等其它 `TextPromptModal` 调用方。

### 不包含范围

- 底部消息输入框本身的行为变更。
- 富文本/Markdown 所见即所得编辑（仍为纯文本编辑）。
- 编辑 assistant / tool 消息（若当前不支持则不在本期扩展）。
- iOS 专项适配（若与 Android 共用组件可顺带一致，但不单独列验收）。
- Core/CLI/Web 消息编辑 UI。

## 核心需求

1. **多行可视区**：编辑弹窗正文输入区默认明显高于单行（建议不少于 4 行等价高度或 ≥120dp），随文本自动增高。
2. **长文滚动**：极长内容时输入区有最大高度，内部滚动，弹窗整体仍可操作「取消/保存」。
3. **回车 = 换行**：与 `ChatComposer` 一致，回车不提交、不优先收起键盘。
4. **保存路径不变**：仅点击「保存」提交；保存中禁用重复点击；失败仍 toast，不自动关弹窗。
5. **打开时体验**：打开编辑时聚焦输入、光标在文末或全文可选中（保持现有 `autoFocus` 行为即可）。
6. **其它弹窗不受影响**：项目/会话重命名等仍保持单行短文本形态。

## 验收标准

### 布局与浏览

- [ ] **A1** 打开含单行长句（>40 字）的用户消息编辑，输入区为多行展示，无需单行横向滚动即可阅读全文。
- [ ] **A2** 打开含 ≥8 行换行的消息，输入区至少显示约 4 行以上可见区域，其余可在输入区内滚动查看。
- [ ] **A3** 弹窗在常见竖屏高度下，「取消」「保存」始终可见、可点，不被输入区挤出屏幕外。

### 键盘与回车

- [ ] **B1** 编辑框聚焦后，按软键盘 **回车** 一次，文本增加一个换行符，**键盘未因该次回车而关闭**。
- [ ] **B2** 连续按回车 3 次，出现 3 个换行，无自动触发保存或关闭弹窗。
- [ ] **B3** 仅点击「保存」后，消息更新、弹窗关闭；与改前 API/DB 行为一致（含 trim 规则若已有则不变）。

### 回归

- [ ] **C1** 会话重命名、新建项目等其它 `TextPromptModal` 仍为单行短输入，无意外变高/多行。
- [ ] **C2** Android 系统返回键：编辑弹窗打开时仍优先关闭弹窗（与现有 `messageEditOpen` 返回处理一致）。
- [ ] **C3** 保存空白或仅空格仍不可保存（按钮置灰或无效）。

### Given / When / Then（主路径）

- **Given** 用户消息正文为多段纯文本  
  **When** 长按 → 编辑 → 在中间某段按回车插入换行 → 保存  
  **Then** 列表中该消息显示新换行，且仅保存一次。

- **Given** 编辑弹窗已打开  
  **When** 用户只按回车、不点保存  
  **Then** 弹窗不关闭、消息列表未变更。
