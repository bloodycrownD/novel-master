# Mobile 聊天稳定性与体验修复 PRD

> **平台**：Android（本迭代验收范围）  
> **组织方式**：单 PRD 大包（滚动 / 流式 / 键盘 / 富文本根因 / 思维链 / 服务商崩溃 / 工具并发评估）  
> **关联迭代**：`mobile-worktree-vfs-perf`（工作区 ↔ 聊天切换）、`mobile-llm-streaming`、`mobile-chat-thinking-and-vfs-sort` 等；本 PRD ** supersede ** 其中未关闭的「列表闪跳、流式抖动」类体验债（若冲突以本文件为准）。

## 背景

Android Mobile 在 **工作区与聊天 Tab/子视图切换**、**流式输出**、**输入框/键盘交互** 场景下，用户反馈明显的 **列表跳动、闪屏、高度抖动**；关闭助手 **富文本（MD/HTML）** 后现象大幅减轻，怀疑 `RenderHTML` 与频繁重排是主因之一，但需在实现前 **先定位根因再定策略**（不预先绑定「关 HTML」或「仅优化布局」）。

此外存在独立 **P0 稳定性** 问题：**服务商页保存 API Key 后异常退出**；以及 **思维链** 目前为纯 `Text`，与 assistant 正文的 Markdown 体验不一致；**工具调用并发** 需产品侧记录评估目标，技术方案后续 SPEC 展开。

## 目标（含成功指标）

| 目标 | 成功指标 |
|------|----------|
| 切换无闪跳 | 工作区 ↔ 聊天（含 session 对话视图）切换时，**恢复离开时的滚动位置与分页状态**，不出现「先显示分页顶部再跳到其它位置」的可见跳动 |
| 流式稳定 | 流式输出时：用户在 **底部附近** 自动贴底且列表不抖；**上翻读历史** 时不强制滚回底部 |
| 键盘稳定 | 键盘弹起/收起、输入框聚焦、编辑消息时：**列表位置稳定**；被编辑气泡保持可见；不因整列表高度重算而反复「顶飞」 |
| 富文本根因 | 完成根因分析（HTML 渲染 / FlatList / 状态重置等），选定策略并落地；**主观**：团队认为开启富文本时抖动可接受，或明确默认策略 |
| 服务商保存 | 保存 API Key **不得** 闪退/异常退出；失败时有可读错误提示 |
| 思维链展示 | 思维链（thinking）与 assistant 正文 **同级 Markdown** 能力（代码块、列表、粗体等） |
| 工具并发 | PRD 记录评估结论与边界；是否并行、顺序保证在 SPEC 定义 |

**整体成功指标（用户选择）**：团队/用户 **主观认定「无明显闪跳、流式与键盘场景可用」**；辅以下文 **验收标准** 手工用例清单。

## 用户与场景

| 用户 | 场景 |
|------|------|
| 日常对话用户 | 在工作区改文件后切回聊天继续对话；期望 **回到刚才阅读位置**，不闪跳 |
| 流式阅读用户 | 看模型逐字输出；在底部时贴底；上翻历史时不被拽回底部 |
| 长文/富文本用户 | 开启助手 MD/HTML；消息含代码块、列表；期望 **不抖** 或根因修复后可开 |
| 编辑消息用户 | 在输入框输入或编辑历史消息；键盘弹出时气泡与列表 **不来回跳** |
| 配置服务商用户 | 在服务商页填写并 **保存 AK**；期望保存成功或明确报错，**不闪退** |
| 工具密集型 Agent 用户 | 一轮回复含多次 tool call；期望执行效率合理、UI 不竞态（并发策略 SPEC 定） |

## 范围

### 包含范围

1. **聊天列表滚动与状态缓存**
   - Tab/子视图（工作区 ↔ 聊天会话）切换时，**按 session 缓存**：滚动 offset（或等价锚点）、是否处于「贴底」态、已加载分页范围（与现有 `loadOlderMessages` 分页一致）。
   - 再次进入同一会话时 **恢复** 上述状态，禁止先渲染错误锚点再 `scrollToEnd` 造成的闪跳。
   - 新建会话或无缓存时，行为可定义（如默认贴底），须在验收中写明。

2. **流式输出滚动策略**
   - **贴底规则**：用户 scroll 距底部 ≤ 阈值（与现 `NEAR_BOTTOM` 语义对齐）时，流式增量 **跟随贴底**；更新方式应避免每 token 触发整表剧烈重排。
   - **非贴底规则**：用户上翻读历史时 **不** 自动 `scrollToEnd`。
   - 流式 tail（`streamingText` / `streamingThinking`）与持久化消息切换时 **无可见跳动**。

3. **键盘与输入/编辑**
   - `ChatComposer`、消息编辑弹层与 `MessageList` 协同：`KeyboardAvoidingView` / inset 变化时 **稳定滚动**。
   - 编辑某条消息时，该气泡 **保持可见**；输入框聚焦不导致整列表反复跳变。

4. **富文本 / HTML 抖动（先调查后决策）**
   - 对比：**富文本开/关**、纯文本、流式开/关 下的复现路径与性能特征。
   - 产出 **根因结论**（如 `RenderHTML` 重挂载、`contentWidth` 变化、`richRenderEpoch`、FlatList `key` 等）与选定策略（优化渲染 **或** 默认策略调整）；策略在 SPEC 锁定，PRD 要求 **结果可感知**。

5. **思维链 Markdown**
   - `ThinkingBlockCard`（及流式 thinking 展示）与 assistant 气泡 **同等 Markdown 渲染能力**（复用现有 rich 管线或等价，受同一 pref/策略约束）。

6. **服务商保存 API Key 崩溃（P0）**
   - `ProviderCreateScreen` / `ProviderEditScreen` / `ProviderForm` 保存路径：**不得** 未捕获异常导致进程退出。
   - 保存成功返回上一页或刷新状态；失败 Toast/Alert，可重试。

7. **工具调用并发（评估项）**
   - 梳理 agent 一轮多 tool 的当前执行模型（串行/并行）。
   - PRD 交付：**评估结论**（是否改并行、UI 展示、错误处理边界）；具体实现列入 SPEC，**本 PRD 不预设技术方案**。

### 不包含范围

- iOS 验收（可后续跟 port）。
- Web / CLI UI。
- 修改 LLM 协议或 provider 后端 API 语义。
- 全新聊天功能（搜索、引用回复等）。
- Token 计数、压缩策略（属 `model-aware-token-counting`）。
- 工作区 VFS 性能优化本身（除非与切换聊天时的 **列表状态** 直接相关）。

## 核心需求

1. **切换恢复位置**：离开聊天会话视图时保存滚动与分页缓存；返回时 **直接** 恢复到离开态，无「先顶后底」闪跳。
2. **流式双模式滚动**：底部跟随 + 上翻不打扰；流式过程中列表 **主观稳定**。
3. **键盘不抖**：输入/编辑/键盘显隐时列表与被编辑内容 **位置稳定**。
4. **富文本根因先行**：先定位 HTML/MD 渲染与抖动的关系，再实施修复；不得在未调查情况下仅关闭功能敷衍。
5. **思维链 Markdown 对齐**：thinking 与 assistant 正文同级 Markdown 体验。
6. **保存 AK 零闪退**：服务商配置保存路径 P0 稳定。
7. **工具并发评估交付**：文档化现状与目标，供 SPEC 实现。

## 验收标准

### 滚动与切换

- **Given** 用户在会话 A 中上翻历史（非底部），已加载若干分页  
  **When** 切到工作区再切回会话 A  
  **Then** 列表 **恢复** 到离开时的可见区域与分页范围；**无** 先显示错误位置再跳的闪动。

- **Given** 用户在会话 A 底部阅读最新消息  
  **When** 切换 Tab 后返回  
  **Then** 仍在底部附近；**无** 先跳到列表顶部再滚底的闪动。

- **Given** 首次进入无缓存的新会话  
  **When** 打开对话  
  **Then** 行为符合 SPEC 定义（如默认贴底）；**无** 红屏。

### 流式

- **Given** 用户在底部，流式输出进行中  
  **When** 持续接收 text/thinking delta  
  **Then** 新内容可见且 **主观无剧烈抖动**；自动贴底。

- **Given** 用户上翻至历史中间  
  **When** 流式输出继续  
  **Then** **不** 自动滚回底部；当前阅读位置保持稳定。

- **Given** 流式结束并持久化为正式消息  
  **When** 列表更新  
  **Then** **无** 明显位置跳变或高度闪动。

### 键盘与编辑

- **Given** 用户在会话中聚焦输入框  
  **When** 键盘弹出/收起  
  **Then** 消息列表 **不** 反复上下跳动；输入框可用。

- **Given** 用户编辑某条历史消息  
  **When** 键盘弹出  
  **Then** 被编辑气泡 **保持可见**；**无** 连续多次 scroll 抖动。

### 富文本

- **Given** 助手富文本 pref **开启**  
  **When** 执行切换/流式/键盘验收用例  
  **Then** 抖动 **≤ 关闭富文本时的主观基线**（或 SPEC 记录的量化阈值）；根因与策略 **有文档**。

- **Given** 富文本 pref **关闭**  
  **When** 同上  
  **Then** 仍满足滚动/流式/键盘验收（回归）。

### 思维链

- **Given** 消息含 thinking 块，内容为 Markdown（含代码块或列表）  
  **When** 展开「思考过程」  
  **Then** 渲染效果与 assistant 正文 **同级**（非纯 monospace 整段 Text）。

- **Given** 流式 thinking  
  **When** 增量更新  
  **Then** 卡片可扩展且 **不** 导致整列表剧烈抖动（与流式验收一致）。

### 服务商（P0）

- **Given** 服务商创建或编辑页，填写合法 baseUrl 与 **新 API Key**  
  **When** 点击保存  
  **Then** **不** 进程退出；成功则返回或刷新；失败则可见错误信息。

- **Given** 保存失败（如校验错误、存储失败）  
  **When** 用户重试  
  **Then** App 仍稳定运行。

### 工具并发（评估交付）

- **Given** 本迭代结束  
  **When** 查阅 KB/SPEC 附件  
  **Then** 存在 **工具并发评估结论**（现状、风险、是否改并行、验收草案）；若实现并行，须有对应自动化或手工用例（SPEC 定义）。

### 回归

- **Given** 本迭代改动合并  
  **When** `npm test -w @novel-master/mobile` 与既有 Android 关键手工路径  
  **Then** **全绿/通过**；无消息丢失、发送、agent run 回归。

## 约束与依赖

- 现有组件：`MessageList`（FlatList）、`ChatTabScreen`（`chatSubview`）、`RichContentBody`（`RenderHTML`）、`ThinkingBlockCard`、`ProviderForm`、agent run / tool 执行链。
- 富文本默认 **关**（`chatRichText` pref）；本迭代不改变默认值，除非调查结论 + 产品确认写入 SPEC。
- Android 为主开发/验收环境。

## 风险与待确认项

| 项 | 说明 |
|----|------|
| FlatList 缓存与 inverted 列表 | 恢复 scroll 可能与 `maintainVisibleContentPosition` 等 RN API 相关，SPEC 选型 |
| 富文本性能上限 | 超长消息已有 `isRichContentOverLimit` 降级；thinking 是否共用同一限制待 SPEC |
| 服务商崩溃复现 | 若需特定机型/步骤，实现阶段补充复现录像到 SPEC |
| 工具并发与 core agent | 并行可能触及 `@novel-master/core` agent-runner，边界在 SPEC 与 core 架构对齐 |

## 里程碑（可选）

1. **调查 + SPEC**：滚动缓存方案、富文本根因、工具并发评估（1 个 design-proposal）。  
2. **P0**：服务商 AK 保存崩溃修复。  
3. **体验主包**：滚动缓存、流式/键盘、思维链 Markdown、富文本策略落地。  
4. **验收**：Android 手工清单 + mobile 测试绿。
