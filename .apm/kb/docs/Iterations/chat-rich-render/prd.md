# 富文本渲染（聊天开关 + 工作区 .md）PRD

> **上游**：`mobile-fix-v2`（聊天纯文本、display 正则、工作区「流式输出」开关已合入 `main`）。  
> **本期**：① 聊天**助手消息**可选 MD+HTML（工作区开关，默认关）；② 工作区 **`.md` 预览始终 Markdown**（与现网一致，不跟开关），并**增强**正文内 HTML/CSS 能力。

## 背景

- 聊天助手回复当前为 **纯文本**，模型输出的 Markdown / HTML 无法按排版展示。
- 工作区 **`.md` / `.markdown`** 在预览模式下**已经**使用 `react-native-markdown-display` 渲染（`FileMarkdownPreview`：Front Matter 卡片 + 正文 Markdown）；**非 .md** 为纯文本。**不依赖**任何「富文本开关」。
- 现网 Markdown 预览对正文内嵌 HTML、`<style>`、`class` 等**增强样式**支持不足；本期在**保持「.md 始终可预览」**前提下升级解析能力，并与聊天侧尽量**共用渲染组件**（实现细节不在本 PRD）。
- 聊天侧仍需**可关闭**的开关，避免长会话默认承担解析成本。

## 目标（含成功指标）

| 目标 | 成功指标 |
|------|----------|
| **视觉还原**（首要） | 聊天（开关开）与工作区 `.md` 样例中，HTML/CSS 在真机排版与预期一致 |
| 聊天默认可关 | 开关**默认关闭**；关时助手消息为纯文本（仍走 display 正则） |
| 工作区 .md 恒开 | **无论开关**，`.md` 预览均为 Markdown（行为不低于现网）；开关**不**关闭 .md 预览 |
| 范围聚焦 | 聊天仅**助手**消息受开关控制；**用户**消息纯文本 |

## 用户与场景

| 用户 | 场景 |
|------|------|
| 日常对话用户 | 助手返回带格式内容，开开关后可读 |
| 工作区写作用户 | 预览笔记 `.md`，始终有 Markdown；HTML 片段样式正确 |
| 注重性能用户 | **关聊天开关**，长会话与现网一致；工作区 .md 预览仍可用 |
| 调试用户 | 关开关看聊天源码；工作区 .md 仍走预览（与编辑切换） |

## 范围

### 包含范围

1. **工作区配置（仅管聊天）**
   - 「我的 → 工作区」增加开关（如「富文本消息」），**默认关闭**，持久化（与 `llmStream` 同级）。
   - **仅控制**：聊天**助手**消息是否 MD+HTML 渲染。
   - **不控制**：工作区 `.md` 文件预览（见下）。

2. **聊天 — 助手消息**
   - **开**：Markdown + HTML（含 CSS，尽量完整，见风险节）。
   - **关**：纯文本（与现网一致）。
   - 管道：**DB 原文 → display 正则 →（开）MD/HTML 渲染**。
   - **流式**：生成中纯文本；结束后刷新再渲染。

3. **工作区 — `.md` / `.markdown` 预览（不跟开关）**
   - 入口：文件编辑器 **预览模式**（`FileMarkdownPreview`）；其他 VFS 阅读入口若接入同一组件则一并受益。
   - **始终**：路径为 `.md` / `.markdown` 时进入 Markdown 预览（Front Matter + 正文），与现网一致；**开关关也不改为纯文本**。
   - **本期增强**：在始终预览的前提下，升级正文对 **HTML 标签与 CSS** 的支持（与聊天「开」时目标能力对齐，**共用渲染器**为宜）。
   - **非 .md**：仍为纯文本（现网），本期不强制改。

4. **验收样例**
   - 聊天助手样例 1 组（受开关约束）。
   - 工作区 `.md` 样例 1 个（**不依赖**聊天开关，预览即验）。

### 不包含范围

- 用开关关闭工作区 `.md` Markdown 预览
- 用户聊天消息富文本
- 文件编辑态 WYSIWYG
- `llm` 通道 / Prompt 格式
- 三档模式；聊天仅 **关 / 开** 两档
- 流式实时富文本
- CLI 展示

## 核心需求

1. 工作区开关**默认关**，**只**管聊天助手 MD+HTML；关时不解析聊天富文本。
2. 工作区 **`.md` 预览恒开 Markdown**（现网行为保持）；开关状态**不影响**是否预览。
3. `.md` 预览本期**增强** HTML/CSS，建议与聊天共用渲染组件；Front Matter 逻辑不变。
4. 聊天助手：开时先 display 正则再渲染；用户消息纯文本；流式结束后渲染。
5. HTML 尽量完整；安全策略实现阶段定。
6. 双端样例验收，首要指标为视觉还原。

## 验收标准

### 开关与默认（仅聊天）

- **Given** 首次使用，**When** 打开「我的 → 工作区」，**Then** 富文本开关为**关闭**。
- **Given** 开关关闭，**When** 查看含 `<div style="…">` 的助手消息，**Then** 气泡为**原始字符**，display 正则仍生效。

### 工作区 .md（与开关无关）

- **Given** 富文本开关**关闭**，**When** 在编辑器打开 `.md` 并进入**预览**，**Then** 仍显示 Markdown 排版（标题/列表等），**不是**整篇纯文本源码；行为不低于现网 `main`。
- **Given** 验收用 `.md` 样例（含内嵌 HTML/CSS），**When** 预览，**Then** 正文 HTML/CSS 达到本期增强标准（与聊天开关无关）。
- **Given** 非 `.md` 文件，**When** 预览，**Then** 仍为纯文本（现网）。

### 聊天 — 助手（开关开）

- **Given** 开关开且有聊天样例，**When** 查看助手消息，**Then** MD 结构可读，HTML/CSS 与预期一致。
- **Given** 开关开，**When** 查看用户消息含 MD/HTML，**Then** 仍为纯文本。

### 管道与流式（聊天）

- **Given** display 规则命中且开关开，**When** 查看助手消息，**Then** 先正则再富文本。
- **Given** 开关开且流式未完成，**When** 生成中 **Then** 纯文本；**When** 结束刷新 **Then** 富文本。

### 性能

- **Given** 开关关，**When** 滚动 50+ 条助手消息，**Then** 较 `main` 无系统性新增卡顿（工作区 .md 预览允许既有 Markdown 成本）。

## 约束与依赖

- 现网参考：`FileMarkdownPreview` + `isMarkdownPreviewPath`（`.md` / `.markdown`）→ `Markdown` 组件；否则 `Text`。
- 依赖：`MessageList`、display 正则、`llmStream` 偏好模式。
- 仅 RN 移动端；真机 Android + iOS 各测聊天（开/关）与工作区 `.md` 各一遍。

## 非功能需求（业务/体验）

- 关开关只影响聊天助手；工作区写 `.md` 用户不因关开关失去预览能力。
- 超长聊天消息 / 大 `.md` 可回退，不改变默认关行为。

## 风险与待确认项

| 项 | 说明 |
|----|------|
| **安全** | 本地 .md 与模型聊天内容均可能含 HTML；消毒**待实现评审**。 |
| **性能** | `.md` 始终 Markdown + HTML 增强可能加重预览成本；大文件回退**待实现**。 |
| **class 样式** | 主题映射 vs 库默认，**待 UI/实现确认**。 |
| **VFS 列表直读** | 若工作区面板未来增加「不经过编辑器」的 .md 阅读，应复用 `FileMarkdownPreview`（实现约束）。 |
| **与正则** | 仅聊天走 display 正则；`.md` 文件原文直接渲染。 |
