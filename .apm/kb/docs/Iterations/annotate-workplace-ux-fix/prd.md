# annotate-workplace-ux-fix PRD

## 背景

本迭代合并三类用户可见问题与能力补齐，均建立在现网已交付能力之上：

1. **文件预览批注高亮**  
   `annotate-cross-node-highlight`（1.4.05）已支持「同单元格内跨行内节点」的下划线。当时文档将「表单元格切断、跨格不硬拼」写成产品边界；实际意图是：**跨单元格 / 整行 / 整表划词时，也应尽量呈现视觉上连续的一整条（或一块）高亮**。本项按**原意图补齐**，不视为改需求。

2. **Mobile 文件编辑工具栏文件名对齐**  
   打开文件进入 FileEditor 后，工具栏中间的文件名（或「未保存」）在预览/未聚焦态不再居中，编辑聚焦时又居中，体验不一致。

3. **常驻工作区（workplace）提示契约**  
   现网 `prompts.workplace` 为布尔开关：开启后向模型注入「user 侧文件树快照 + assistant 侧固定 `【done】`」；user 侧无外层包裹标签；编辑器仅有开关与说明，**没有**像系统提示词那样的可编辑输入框。产品希望：  
   - 常驻 user 内容用 `<workplace>…</workplace>` **仅作包裹**（与历史「回复增量里的 workplace」无关）；  
   - assistant 确认语改为可配置字符串；  
   - UI 对齐系统块：开启后提供输入框，且**禁止为空**；默认预填表达「我已看到工作区」的文案；  
   - 旧配置 `workplace: true` 仍可加载，语义兼容为 assistant 返回 `true` / `done` 一类确认。

## 目标（含成功指标）

| 目标 | 成功指标 |
|------|----------|
| 表格跨格划词可见连续高亮 | 在预览态对跨单元格 / 整行 / 整表连续选区添加批注后，下划线（或等价标记）在视觉上尽量连成一条/一块，而非仅同格有效 |
| 工具栏文件名对齐 | Mobile FileEditor 在预览、未聚焦编辑、聚焦编辑、脏「未保存」各态下，中间文件名在工具栏槽内均居中 |
| 常驻工作区可配确认语 | Agent 编辑器开启常驻工作区后可编辑助手文案；运行时 assistant 侧使用该文案（非写死 `【done】`） |
| 常驻内容可识别包裹 | 注入模型的常驻 user 正文被 `<workplace>…</workplace>` 包裹 |
| 存量兼容 | 仅含 `workplace: true` 的旧 Agent 配置无需手工改 YAML 即可继续开启并工作 |

## 用户与场景

| 用户 | 场景 |
|------|------|
| 写作者 | 在含 Markdown 表格的会话文件预览中划过整行/多格文字并添加批注，需要看到连续下划线以便回看与点开 |
| Mobile 写作者 | 打开/预览文件时靠工具栏文件名辨认当前文件，期望名称始终居中 |
| Agent 配置者 | 为写作 Agent 打开常驻工作区，自定义助手确认语（默认表达「已看到工作区」），并希望发出去的工作区快照有明确 XML 包裹 |
| 存量配置维护者 | 升级后旧 `workplace: true` 的 Agent 仍可用，不必立刻改成字符串 |

## 范围

### 包含范围

1. **批注预览高亮（表格）**  
   - 跨单元格、整行、整表（用户一次连续划词所能覆盖的表格区域）上，批注下划线**尽量视觉连续**。  
   - Desktop 与 Mobile 文件预览批注路径行为一致（同合同）。  
   - 仍保留「尽力匹配」底线：匹配失败时批注条目与 Composer chip 可保留，但本迭代以「表格连续选区应能亮」为成功标准。

2. **Mobile FileEditor 标题/文件名对齐**  
   - 修复工具栏中间 basename /「未保存」在各常用态下不居中的问题。  
   - 范围限于 FileEditor 屏内工具栏文件名，不要求改 AppHeader 固定文案「编辑文件」。

3. **常驻工作区配置与呈现**  
   - Schema：由「仅布尔」升级为可表示「关闭 / 开启且带助手确认语文案」的形态；开启时文案为字符串（示例默认：`i have seen workplace`）。  
   - **兼容**：存量 `workplace: true` 视为开启，助手确认语按 `true` 或 `done` 均可接受的兼容语义处理（产品上二者等价于「已确认看到工作区」的旧行为桥接）。  
   - **注入**：常驻 user 侧文件树等内容须包在 `<workplace>…</workplace>` 内；该标签**只表示包裹常驻前缀**，**不**表示回复增量、**不**恢复已废止的消息内增量 `<workplace>` 语义。  
   - **UI**（Desktop / Mobile Agent 编辑器）：对齐系统提示词块——开关开启后出现输入框；预填 `i have seen workplace`；**禁止保存空文案**（开启态下输入框不得为空）。

### 不包含范围

- 重新引入消息正文 / 回复增量中的 `<workplace>` 增量协议（与 `annotate-user-ops-unify` 废止结论一致）。  
- 改变工具回合（tool turn）桥接文案是否跟常驻配置联动（默认**不**纳入本迭代；若实现期发现强耦合，记入待确认后再扩）。  
- 跨文件、非连续多选区的批注高亮。  
- Desktop FileEditor / 非 Mobile 工具栏的同类对齐（除非自然共用同一组件且零成本）。  
- 项目级 Agent、全局 Agent 以外的新入口；不改 VFS 规则引擎语义本身。

## 核心需求（3-7 条）

1. **表格连续选区高亮**：用户在预览中对跨单元格 / 整行 / 整表做一次连续划词并添加批注后，预览中应尽量呈现**视觉上连续**的一整条或一块高亮（下划线或等价标记）。  
2. **双端一致**：同类表格选区在 Desktop 与 Mobile 预览批注上，高亮合同一致。  
3. **FileEditor 文件名居中**：Mobile 打开文件后，工具栏中间文件名（及脏态「未保存」）在预览与编辑相关各态均居中对齐。  
4. **常驻工作区可配助手文案**：开启时常驻注入的 assistant 侧使用配置字符串；默认预填 `i have seen workplace`；开启态禁止空串。  
5. **`<workplace>` 包裹**：常驻 user 侧注入正文外层使用 `<workplace>…</workplace>` 包裹，且仅表示常驻前缀，不表示增量回复。  
6. **布尔兼容**：旧 `workplace: true` 加载后仍为「开启」，助手确认语兼容理解为 `true` / `done` 一类确认，无需用户立刻改配置即可使用。

## 验收标准

### 批注 · 表格连续高亮

- **Given** 会话工作区预览打开含 GFM 表格的 Markdown  
  **When** 用户连续划选**跨两个及以上单元格**的可见文本并添加批注  
  **Then** 预览中高亮在视觉上尽量连成一条/一块（允许实现上多段标记，但观感须连续，不得仅首格有下划线）。

- **Given** 同上  
  **When** 用户划选**整行**或**整表**可见连续文本并添加批注  
  **Then** 高亮覆盖该次选区对应的可见范围，且观感连续。

- **Given** Desktop 与 Mobile  
  **When** 对同一表格文件做同类跨格划词批注  
  **Then** 两端均满足上述高亮合同。

### Mobile FileEditor 对齐

- **Given** 用户打开任意文件进入 FileEditor（默认预览或未聚焦编辑）  
  **When** 查看工具栏中间文件名  
  **Then** 文件名在中间槽内居中（非明显偏左）。

- **Given** 进入编辑并聚焦、或内容已脏显示「未保存」  
  **When** 查看工具栏中间文案  
  **Then** 仍保持居中，与预览态一致。

### 常驻工作区

- **Given** 用户在 Agent 编辑器打开「常驻工作区」开关  
  **When** 首次开启  
  **Then** 出现输入框且预填 `i have seen workplace`；若清空则无法成功保存（开启态空文案须在表单构建入口显式阻断，不得静默写出空串）。

- **Given** 已配置非空助手文案并开启常驻工作区  
  **When** 在会话中触发常驻注入（有可注入的工作区内容时）  
  **Then** 发给模型的 assistant 侧确认语等于所配文案；user 侧常驻正文被**恰好一层** `<workplace>…</workplace>` 包裹（不得双包）。

- **Given** 存量 Agent 配置为 `workplace: true`（无字符串）  
  **When** 升级后加载并运行  
  **Then** 仍视为开启常驻工作区；助手确认语按兼容语义（`true` / `done` 均可接受的旧确认）工作，不因缺字符串而崩溃或静默关闭。

- **Given** 用户或文档提及 `<workplace>`  
  **When** 配置或查看常驻注入  
  **Then** 该标签仅出现在常驻前缀包裹场景，**不**作为回复增量标签恢复。

## 风险与待确认项

- **「视觉连续」的实现边界**：跨格时浏览器/WebView 选区串可能含制表符或换行；验收以「观感连续、覆盖本次选区」为准，精确像素级连笔不强制。  
- **换行归一的已知代价**：为匹配跨格选区，实现会去掉 needle 中的制表符/换行；因此 `<pre>` 或单文本节点内含换行的选区**可能不再命中**。本迭代验收**接受**该限制，优先保证表格连续选区的观感连续。  
- **默认预填文案语言**：已定为英文 `i have seen workplace`（含义：我已看到工作区）；若上线前需改为中英双语整句，可再改一版文案，不改本 PRD 结构。  
- **工具回合 bridge**：现网工具回合亦使用 `【done】` 类桥接；本迭代默认**不**要求与常驻助手文案联动，若联调发现必须统一，再单开确认。  
- **误标权衡**：为追求跨格连续高亮，须避免把不相邻格内容错误拼进同一命中；若连续与零误标冲突，优先「覆盖本次连续选区」且不得高亮未选中的无关段落。
