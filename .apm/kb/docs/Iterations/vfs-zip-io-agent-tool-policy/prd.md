# VFS ZIP 导入导出与 Agent 工具名单 PRD

> **范围**：同一迭代、一份 PRD；Core + CLI + RN 移动端语义一致。  
> **边界**：本 PRD 不含技术方案、接口、表结构、任务拆分（见后续 SPEC）。

## 背景

- Novel Master 已在 **global / project / session** 三域提供 VFS 文本文件能力（模板与会话工作区），但缺少**批量备份、迁移、跨设备同步**的便携格式。
- Agent 运行时会向模型暴露 **ToolRegistry** 中已注册工具；当前无法按 Agent **限制可用工具集**，不利于安全收敛或专用 Agent（如只读、只写）。

## 目标（含成功指标）

| 目标 | 说明 |
|------|------|
| VFS 便携 IO | 三域各自支持 **ZIP 导出 / 导入**，内容为**当前可见文件树的纯文本快照** |
| 导入语义清晰 | 导入 = **该域工作区完全替换**为 ZIP 内文件树（ZIP 即真相源） |
| Agent 工具策略 | Agent 定义支持 **白名单或黑名单二选一**；默认未配置 = **全部已注册工具可用** |
| 三端一致 | Core 实现；`nm` CLI 与移动端提供等价能力（入口可不同，行为可判定一致） |

**成功指标（可量化）**

- 三域各至少 **1 条**自动化或 CLI 文档用例：导出 ZIP → 篡改 → 导入 → 域内文件与 ZIP 一致（± 明确排除项）。
- Agent：至少 **6 条**可判定用例（默认全工具、白名单、黑名单、空白名单、互斥校验、对 LLM 暴露工具列表一致）。
- `npm test`（core + cli）与移动端相关用例在迭代完成时通过。

## 用户与场景

| 用户 | 场景 |
|------|------|
| 写作用户（移动端） | 备份当前会话工作区；从同事处导入 session ZIP 覆盖恢复 |
| 模板维护者 | 导出/导入 **全局** 或 **项目** 模板域 ZIP，批量维护 `/` 下文件（如 `/seed/prompt.md`） |
| 开发者 / 运维 | CLI 脚本化导出种子数据、导入到指定 project/session |
| Agent 配置者 | 为「只读分析」Agent 配白名单仅 `vfs.read`/`vfs.grep`；为通用写作 Agent 保持默认全启用 |

## 范围

### 包含范围

**1. VFS ZIP 导出（按域单独操作）**

- **global 域**：导出该域**当前有效**文本文件（逻辑路径以 `/` 为根）。
- **project 域**：导出指定 project 模板域**当前有效**文本文件（逻辑路径以 `/` 为根）。
- **session 域**：导出指定 session 工作区**当前有效**文本文件（逻辑路径以 `/` 为根）。
- 每次导出产生 **一个 ZIP**，**仅包含一个域**的文件树（不混域）。
- ZIP 内路径与域内逻辑路径一致（如 session 导出含 `/notes.md`，不含 `projects/…` 等跨域前缀）。
- **仅文本文件**：以 **UTF-8** 文本读写；**不包含**二进制、规则、状态、KKV、消息、快照元数据、执行记录、版本号等。

**2. VFS ZIP 导入（按域完全覆盖）**

- 针对选定域执行导入：导入完成后，该域 VFS **仅保留 ZIP 中出现的文件路径与内容**（等价于用 ZIP 快照**完全替换**该域当前文件树）。
- ZIP 中不存在的路径：导入后在该域**不再存在**（删除域内多余文件）。
- 冲突策略：**不做合并、不跳过**；以 ZIP 为准一次性替换。
- 导入前须有**明确确认**（移动端对话框 / CLI `--yes` 或交互确认），避免误覆盖。
- **两阶段、先验后写**：先在内存中完整解析并校验 ZIP（路径合法、条目均为可读 UTF-8 文本、无非法路径等）；**任一校验失败则整包拒绝，不改动该域工作区**。
- **写入阶段事务性**：校验通过后，再对该域执行全量替换；若写入过程中失败，**整域回滚**至导入前快照，域内状态与导入前一致。

**3. 交付面**

- **Core**：域级 export/import 能力（供 CLI、Mobile 调用）。
- **CLI**：`nm` 子命令（或等价）支持三域 export/import。
- **RN 移动端**：在对应工作区/模板入口提供导出、导入；**导出**为系统「另存为」保存 ZIP 到用户选择的本地路径，**导入**为系统文件选择器选择 ZIP。

**4. Agent 工具白名单 / 黑名单**

- 配置写在 **Agent 定义文件（YAML/JSON）** 中，随 Agent 一起分发与版本管理。
- **白名单**与**黑名单** **二选一**，同一 Agent **不得同时配置**；校验失败时加载/保存 Agent 报错。
- **默认**：未配置任何名单 → **ToolRegistry 中全部已注册工具**对应该 Agent 可用，并暴露给模型。
- **白名单模式**：仅列表内工具名可用；**白名单为空** → **不允许任何工具**（模型侧无工具或等价空集）。
- **黑名单模式**：列表内工具禁用，其余已注册工具可用；**黑名单为空** → 等同未配置（全部可用）。
- 名单条目为工具 **注册名**（与 ToolRegistry `name` 一致），作用于**当前已注册的全部工具**（含未来扩展工具，名称须显式列入方可启用/禁用）。

### 不包含范围

- 单 ZIP 混装多域（global+project+session）；本迭代 **按域分 ZIP**。
- 导入导出消息历史、会话元数据、project/session 实体、正则/压缩/模型等工作区**状态与规则**。
- 非 UTF-8 文本、二进制条目（ZIP 内出现即 **整包拒绝**，不部分导入）。
- Agent 工具名单的 **Registry/UI 覆盖层**（仅以 YAML 为准）。
- 同时配置白名单与黑名单时的运行时优先级规则（直接禁止）。
- Web 端、自动化云同步、增量/差异导入、导入预览逐文件勾选。

## 核心需求

1. **三域独立 ZIP**：global / project / session 均可单独导出、单独导入；ZIP 只含该域文本文件树。
2. **导入即全量替换**：导入后域内文件集合与 ZIP 一致；域内 ZIP 未包含的路径被移除。
3. **纯文本与安全确认**：仅 UTF-8 文本；导入前用户确认；**先内存校验、后写入**；校验失败或写入失败均 **不污染工作区**（写入失败整域回滚）。
4. **CLI 与移动端等价**：同一域、同一 ZIP 在 CLI 导入与移动端导入后，域内文件列表与内容一致。
5. **Agent 默认全工具**：无名单配置时，行为与现网一致（全部已注册工具可用）。
6. **Agent 白/黑名单互斥**：仅允许一种模式；白名单空 = 无工具；配置非法时 Agent 无法加载。
7. **工具暴露与执行一致**：被禁用的工具不出现在发给模型的 tools 定义中，且运行时不应被执行（若模型幻觉调用，应有可判定错误）。

## 验收标准

### VFS — 导出

- **Given** session 域存在 `/a.md`、`/dir/b.md`（均为 UTF-8 文本），**When** 执行 session 域导出，**Then** 得到 ZIP，解压后仅含对应逻辑路径的文本内容，且与域内读取一致。
- **Given** global 域有 `/x.md`，**When** 执行 global 域导出，**Then** ZIP 条目为 `x.md`（无 `template/` 前缀），且不包含 `projects/` 等其它域路径。
- **Given** project 域有 `/prompt.md`，**When** 执行 project 域导出，**Then** ZIP 仅含该 project 文件树（逻辑路径形状与 session 一致）。

### VFS — 导入（完全覆盖）

- **Given** session 域现有 `/old.md`，**When** 导入仅含 `/new.md` 的 ZIP 且用户确认，**Then** 域内仅有 `/new.md`，`/old.md` 不存在。
- **Given** 导入 ZIP 与导出前快照一致，**When** 导入到空域或已有文件域，**Then** 域内文件树与 ZIP 完全一致。
- **Given** 用户未确认导入，**When** 触发导入，**Then** 域内文件无变化。
- **Given** ZIP 含非 UTF-8、二进制或非法路径（如 `..`），**When** 导入，**Then** 内存校验阶段失败、给出明确错误，**且该域工作区与导入前完全一致**。
- **Given** 校验通过但写入过程中发生错误，**When** 导入结束，**Then** 该域工作区已回滚至导入前状态（无「半套旧 + 半套新」）。
- **Given** 校验通过且写入成功，**When** 导入结束，**Then** 该域文件树与 ZIP 完全一致。

### VFS — 三端

- **Given** 同一 session 与同一 ZIP，**When** 分别经 CLI 与移动端导入，**Then** 域内 `list` 结果一致（路径集合相同）。
- **Given** 移动端从会话工作区入口导出，**When** 在 CLI 对同 session 导入该 ZIP，**Then** 内容一致。

### Agent — 工具名单

- **Given** Agent YAML 未配置工具名单，**When** 加载并 run，**Then** 暴露给模型的工具集合 = 当前 ToolRegistry 全部已注册工具。
- **Given** 白名单仅含 `vfs.read`、`vfs.grep`，**When** 加载并 run，**Then** 模型 tools 仅含上述两项；`vfs.write` 不在列表中。
- **Given** 白名单为空数组（或等价空配置），**When** 加载，**Then** 无工具暴露给模型（或 tools 为空，行为可文档化且一致）。
- **Given** 黑名单含 `vfs.write`，**When** 加载并 run，**Then** 除 `vfs.write` 外已注册工具均可用。
- **Given** 同一 Agent 同时配置白名单与黑名单，**When** 保存或加载，**Then** 报错，Agent 不可用。
- **Given** 白名单含 Registry 中不存在的工具名，**When** 加载，**Then** 报错，Agent 不可用。

## 约束与依赖（扩展）

- 依赖现有 **三域 VFS 绑定**（见 [chat-project-vfs PRD](../chat-project-vfs/prd.md)）与 **Tool 系统**（见 [tool-system PRD](../tool-system/prd.md)）、**Agent 定义**（见 [agent-system PRD](../agent-system/prd.md)）。
- 移动端依赖系统文件选择与本地另存为能力；CLI 依赖本地文件路径（`--out` / `--file`）。
- Agent 名单仅 YAML：移动端 Agent 编辑页需支持编辑并写回 YAML 字段（或等价序列化），不得引入第二套持久化真相源。

## 风险与待确认项（扩展）

| 项 | 说明 |
|----|------|
| 大 ZIP / 大目录 | 内存全量解析 + 回滚快照可能占内存；SPEC 需约定上限或进度提示 |
| 隐藏/非法路径 | ZIP 内 `..`、绝对路径等：校验阶段拒绝，不写入 |
| 工具名变更 | Registry 改名后旧 Agent YAML 需人工更新 |
| 无效工具名 | 白/黑名单含未注册工具名：加载 Agent 时报错（验收可判定） |
