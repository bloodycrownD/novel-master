# 项目智能体配置 PRD

## 背景

Novel Master 以 **项目（Project）** 组织会话与项目工作区模板，以 **智能体（Agent）** 定义提示词、可选模型 pin、运行时步数与工具策略。当前：

- **项目**仅包含名称与时间戳，**无**行为配置；切换项目不改变 Agent 行为。
- **智能体**存放在全局注册表；工作区通过单一指针 `currentAgentId` 选择「当前 Agent」，对所有项目生效。
- 用户若希望「小说 A 用写作 Agent、小说 B 用审稿 Agent」，只能手动切换全局 Agent，易误用且无法随项目记忆。

本迭代在 **项目配置** 中增加 **智能体配置**入口：项目可选择 **跟随全局** 或 **使用本项目专属智能体**。专属智能体仅在该项目上下文中可用，不出现在全局 Agent 列表供其他项目选用。

## 目标（含成功指标）

| 目标 | 成功指标 |
|------|----------|
| 项目级 Agent 策略 | 每个项目可配置智能体模式：**跟随** 或 **自定义** |
| 跟随全局 | 模式为跟随时，该项目下会话 Agent 行为与现网「工作区当前 Agent」**一致** |
| 项目专属 Agent | 模式为自定义时，该项目下会话使用项目内配置的智能体；**其他项目不可选用** |
| 配置可发现 | Desktop / Mobile 均能从项目相关入口进入智能体配置 |
| 存量兼容 | 未配置过的存量项目默认 **跟随**；行为与现网 **无差异** |
| 互斥清晰 | 同一项目 **仅** 「跟随」或「自定义」二选一，不可同时生效 |

## 用户与场景

| 用户 | 场景 |
|------|------|
| 多项目写作者 | 项目「玄幻」绑定自定义写作 Agent，项目「随笔」跟随全局默认 Agent；切换项目后自动用对智能体 |
| 配置者 | 在项目设置中为某书单独微调 system 提示词与工具策略，而不污染全局 Agent 库 |
| 协作者 | 打开项目即使用该项目约定的 Agent，减少「忘切换全局 Agent」导致的风格漂移 |
| 维护者 | 全局 Agent 库继续管理通用模板；项目专属 Agent 与全局列表隔离，避免注册表膨胀 |

## 范围

### 包含范围

1. **项目智能体配置入口**
   - 在项目相关设置中新增 **智能体** 配置区（Desktop / Mobile 均需可达）。
   - 配置项：
     - **跟随**（默认）— 使用全局工作区当前 Agent（与现网 `currentAgentId` 解析一致）。
     - **自定义** — 使用仅属于本项目的智能体配置。

2. **自定义智能体**
   - 用户选择「自定义」后，可编辑本项目专属智能体；**编辑界面能力与现网全局 Agent 编辑器一致**（提示词布局、可选模型 pin、运行时步数、工具 allow/deny 等）。
   - 保存后仅绑定当前 `projectId`；**不** 写入全局 Agent 注册表供其他项目选择。
   - 其他项目界面 **不可** 看到或选用该项目的专属 Agent。

3. **运行时解析**
   - 在项目的会话中发起 Agent 时：
     - 跟随 → 与现网相同，解析全局 `currentAgentId` 与全局注册表。
     - 自定义 → 使用该项目持久化的专属 Agent 定义。
   - 切换项目后，下一次 Agent 运行使用 **新项目** 的智能体策略（无需用户再手动切全局 Agent）。

4. **存量与默认**
   - 升级后所有既有项目：智能体模式 = **跟随**。
   - 跟随模式下，用户切换全局 Agent 仍影响该项目（与现网一致）。

5. **与全局 Agent 管理的关系**
   - 全局「Agent 管理」列表与 CRUD **保留**；用于维护通用 Agent 与跟随模式下的来源。
   - 项目自定义 Agent **不** 出现在全局列表；删除项目时一并清理其专属 Agent 配置。

### 不包含范围

- 按 **会话（Session）** 单独覆盖 Agent（仅项目级）
- 项目级 **模型**、**正则组**、**压缩条件** 等配置（可 follow-up 复用同一「项目设置」壳层）
- 项目间 **复制/导入** 专属 Agent 模板
- 云同步冲突策略的完整产品定义（若已启用云同步，实现阶段与存储方案一并考虑，本 PRD 要求：**本地行为正确**）
- 修改 Agent 提示词引擎或 tool 执行语义
- CLI 项目级 Agent 配置子命令（可 follow-up）

## 核心需求

1. **二选一模式**：每项目仅「跟随全局」或「自定义专属」一种生效状态。
2. **默认跟随**：新项目与存量项目默认跟随；与现网行为一致。
3. **专属隔离**：自定义 Agent 仅当前项目可用；全局 Agent 列表与其他项目不可引用。
4. **编辑器复用**：自定义态下配置体验与全局 Agent 编辑器对齐，降低学习成本。
5. **切换项目即切换策略**：用户切换当前项目后，Agent 解析遵循该项目配置，无需额外操作。
6. **双端可达**：Desktop 与 Mobile 均可完成查看与修改项目智能体配置。

## 验收标准

### 默认与跟随

- **Given** 存量项目，用户从未打开项目智能体配置  
  **When** 在该项目会话中运行 Agent  
  **Then** 使用全局当前 Agent，与现网 **一致**。

- **Given** 项目 A 为 **跟随**，用户在工作区将全局 Agent 改为 X  
  **When** 在项目 A 的会话中运行 Agent  
  **Then** 使用 Agent X。

### 自定义专属

- **Given** 用户将项目 B 设为 **自定义**，并配置专属 Agent（名称/提示词与全局 Agent C 不同）并保存  
  **When** 在项目 B 的会话中运行 Agent  
  **Then** 使用项目 B 的专属配置，**不** 使用全局当前 Agent。

- **Given** 项目 B 为自定义，全局当前 Agent 为 X  
  **When** 用户在项目 A（跟随）与项目 B 各运行一次 Agent  
  **Then** A 使用 X；B 使用 B 的专属 Agent。

- **Given** 项目 B 的自定义 Agent 已保存  
  **When** 用户打开全局 Agent 管理列表  
  **Then** 列表中 **不出现** B 的专属 Agent。

- **Given** 项目 B 的自定义 Agent 已保存  
  **When** 用户在项目 C 的智能体配置中查看可选 Agent  
  **Then** **不可** 选用 B 的专属 Agent。

### 模式切换

- **Given** 项目曾为自定义，用户改回 **跟随** 并保存  
  **When** 运行 Agent  
  **Then** 恢复使用全局当前 Agent；专属配置不再生效（是否保留草稿实现阶段定，验收：**跟随态下不使用专属**）。

- **Given** 用户从项目 B 切换到项目 A  
  **When** 立即运行 Agent  
  **Then** 使用项目 A 的智能体策略（跟随或自定义各自生效）。

### 删除与隔离

- **Given** 用户删除项目 B  
  **When** 删除成功  
  **Then** B 的专属 Agent 配置不再可被任何入口访问。

### 双端一致

- **Given** 用户在 Desktop 将项目设为自定义并保存  
  **When** 用户在 Mobile 打开同一项目的智能体配置  
  **Then** 显示为 **自定义**，且内容与 Desktop 保存一致。

## 约束与依赖

- 依赖 `agent-system` Agent 定义结构与注册表、运行时解析。
- 依赖 `chat-project-vfs` 项目实体与会话归属 `projectId`。
- 依赖 `persistent-state-and-preferences` 工作区全局 `currentAgentId` 指针（跟随模式）。
- 与 `model-generation-params`、`desktop-chat-workspace-polish` **无** 功能依赖，建议 **独立迭代**。

## 非功能需求（业务/体验）

- 项目智能体配置页应标明当前模式（跟随 / 自定义），避免用户误以为在改全局 Agent。
- 跟随模式下，可只读展示当前生效的全局 Agent 名称，便于核对。
- 自定义模式下，保存成功有明确反馈。

## 风险与待确认项

| 项 | 说明 |
|----|------|
| 存储形态 | 实现阶段确定项目 settings 落库方式；PRD 要求行为正确、存量默认跟随 |
| 云同步 | 若同步 project 表，专属 Agent 是否同步、冲突策略——实现 SPEC 细化 |
| 删除项目 | 专属配置随项目删除；是否提供导出备份 follow-up |
| 全局 Agent 删除 | 跟随模式下若全局 Agent 被删，行为与现网错误提示一致（如要求重新选择） |
| 自定义模型 pin | 项目专属 Agent 可 pin 模型 id；仍使用全局已保存模型，本迭代不引入项目级模型库 |
