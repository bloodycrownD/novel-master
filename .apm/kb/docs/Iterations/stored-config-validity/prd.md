# 存储配置有效性（Stored Config Validity）PRD

## 背景

近期迭代对 **智能体配置**、**事件配置** 等做了协议升级（如事件 DAG `schemaVersion: 2`、智能体 prompts 三区布局等）。当前产品在 UI 层分散实现了多种「兼容/容错」路径：

- **事件配置**：整份 decode 失败时显示恢复卡片；部分未知 action 时仍可进入编辑器并占位展示，需用户手动移除后才能保存。
- **智能体配置**：列表对单条 decode 失败显示错误标签；编辑页则直接失败。

上述行为目标一致——**告知用户配置已失效，需删除或重置**——但实现分散、分支多，维护成本高。用户不希望继续做 wire 层自动迁移或「边编辑边修」。

同时，`bootstrapNovelMaster` 中累积了大量 **一次性** 的历史 migrate（删废弃列、清废弃 KKV、加列、数据回填等）。对已用近期版本打开过的工作区，这些逻辑多为空操作；团队接受 **不再维护** 这类自动 DB 修补，改由「配置失效 → 用户重置」与「极旧库不支持」的产品立场兜底。

## 目标（含成功指标）

| 维度 | 目标 | 成功指标 |
|------|------|----------|
| **一致性** | 智能体、事件等存储配置共用同一套「有效 / 失效」判定与文案 | Mobile、Desktop 对同一失效样例展示一致的主提示与操作项 |
| **简化** | 去掉事件 unknown action 占位编辑等 lenient 路径 | 失效配置 **不渲染** 正常编辑器；事件侧无「未知动作占位块」「移除未知动作」类交互 |
| **可恢复** | 用户可自行处理失效数据 | 事件：可 **恢复默认并保存** / **清空并保存默认**；智能体：列表可 **删除**，编辑页可 **删除** 或 **用默认模板覆盖（保留 agentId 与显示名称）** |
| **启动精简** | 移除已无必要的 bootstrap migrate | `bootstrapNovelMaster` 仅保留当前 DDL 建表与必要种子数据；遗留 migrate 文件与对应测试删除 |
| **支持边界清晰** | 不承诺打开任意历史 DB | PRD/SPEC 写明：仅保证当前版本新建库 + 已在近期版本成功打开过的工作区 |

## 用户与场景

- **用户**：在 Mobile / Desktop 中维护智能体、事件配置的用户。
- **场景 A**：升级 App 后打开「事件配置」，本地仍为 v1 `parallel`/`sequential` 或含已移除 action → 看到 **配置已失效** 面板，一键恢复默认。
- **场景 B**：某智能体 prompts 含已废弃字段（如 `prompts.blocks`）→ 列表显示 **失效** 标签，可删除；进入编辑页见失效面板，可选择删除或用默认模板覆盖（名称保留）。
- **场景 C**：开发/发布新版本后，bootstrap 启动更快、代码更少，不再执行历史列清理与 KKV purge migrate。

## 范围

### 包含范围

1. **统一配置有效性模型（业务语义）**
   - 对智能体 wire、事件 wire 等：**版本不匹配** 或 **当前 App 无法 strict 解析** → 一律视为 **失效**。
   - **不做** wire 自动迁移、不做「未知字段/未知 action 可编辑占位」。

2. **事件配置（Mobile + Desktop）**
   - 加载时若失效：全页 **失效面板**（不展示事件块编辑器）。
   - 操作：**恢复默认并保存**、**清空旧配置并保存默认**（与现有恢复语义对齐，文案与布局两端统一）。
   - 移除：未知 action toast、占位卡片、「移除未知动作」等交互。

3. **智能体配置（Mobile + Desktop）**
   - **列表**：失效行显示明确标签/说明；支持 **删除**；不提供正常编辑能力（或进入编辑即见失效页）。
   - **编辑页**：失效时 **失效面板**；操作：**删除该智能体**、**用默认模板覆盖（保留 agentId 与当前显示名称）**。
   - 覆盖默认后进入可编辑的正常表单。

4. **Bootstrap migrate 清理**
   - 删除 `packages/core/src/bootstrap` 下历次遗留的一次性 migrate 及在 `novel-master-bootstrap` 中的调用（含列 DROP、废弃表 DROP、KKV purge、列 ADD/RENAME、worktree/prefs 数据搬迁等）。
   - 删除或改写仅服务于上述 migrate 的测试。
   - 保留：当前模块 `CREATE TABLE IF NOT EXISTS` DDL、内置 provider 种子等 **非历史修补** 逻辑。

5. **文案与体验**
   - 失效原因对用户可读（如「配置版本过旧」「含已移除字段」），技术细节可折叠或次要展示。
   - Mobile、Desktop 共用 core/config-forms 侧文案常量（业务一致）。

### 不包含范围

- Wire / JSON **自动迁移**（含压缩条件 KKV v2→v3 读时写回；若需改为失效+重置，单列迭代）。
- SQLite 表新增 `schemaVersion` 列或 DB 世代元数据表（本迭代以 wire 内 `schemaVersion` + strict 解析为准）。
- CLI 交互大改（可沿用 strict decode 报错；列表类命令是否与 UI 对齐见 SPEC）。
- 聊天消息、VFS 文件、Provider 模型列表等非「智能体/事件配置文档」类数据。
- 云同步冲突策略、全库备份格式变更。

## 核心需求

1. **失效即阻断编辑**：智能体、事件配置一旦判定失效，不得展示可保存的编辑表单（事件不得再出现 unknown action 占位编辑）。
2. **统一判定入口**：有效性判断集中在 core（非 Mobile/Desktop 各自 try/catch + 分支），UI 只消费「有效 / 失效 + 原因 + 建议操作」。
3. **事件重置**：失效用户仅能通过 **恢复/清空并保存默认** 恢复可用配置，无第三条「带病保存」路径。
4. **智能体双入口**：列表侧重 **识别 + 删除**；编辑页侧重 **说明 + 删除或覆盖默认（保留 id 与名称）**。
5. **Bootstrap 瘦身**：移除历史 migrate 后，启动路径只负责 **确保当前 schema 存在**；不承诺修补从未被近期版本打开过的极旧数据库文件。
6. **支持边界公示**：发布说明或文档中注明——从过旧备份还原且未曾在中间版本打开的工作区，可能需要新建工作区或重新导入配置。

## 验收标准

### 配置有效性 — 事件

- **E1** Given 本地 `nm-events` 为 v1（`parallel`/`sequential`）或 strict decode 失败，When 打开事件配置页，Then 仅显示失效面板与重置操作，**不**显示事件块列表编辑器。
- **E2** Given 失效事件配置，When 用户点击「恢复默认并保存」，Then 保存成功且再次进入为可编辑的默认 DAG 配置。
- **E3** Given 含已移除 action（如 `refresh-macros`）的 v2 文档，When 打开事件配置页，Then 按 **E1** 处理（失效面板），**不**出现 unknown 占位块。
- **E4** Mobile 与 Desktop 对同一失效样例，主标题与主操作文案一致。

### 配置有效性 — 智能体

- **A1** Given 某 agent wire 含废弃字段导致 decode 失败，When 查看智能体列表，Then 该行有失效标识且可 **删除**。
- **A2** Given 同上，When 进入该智能体编辑页，Then 显示失效面板，提供 **删除** 与 **用默认模板覆盖（保留 agentId 与显示名称）**。
- **A3** Given 用户选择覆盖默认，When 操作完成，Then 再次进入为正常可编辑表单，agentId 与名称不变，prompts/runtime/tools 等为默认模板内容。
- **A4** Given 列表中失效智能体，When 用户仅删除且不进入编辑，Then 列表更新，无崩溃。

### Bootstrap migrate 清理

- **B1** `bootstrapNovelMaster` 不再调用已删除的遗留 migrate；对应 migrate 源文件已从仓库移除。
- **B2** 在当前 `main` 分支典型开发库上执行 bootstrap，应用可正常启动，智能体/事件/聊天等核心路径可用。
- **B3** 全新安装（空库）执行 bootstrap 后，DDL 表齐全，可创建项目与会话。
- **B4** 仅服务于已删 migrate 的测试用例已删除或改写；core bootstrap 相关测试通过。

### 回归

- **R1** 合法的事件 v2、智能体 v1 配置在 Mobile/Desktop 上仍可正常加载、编辑、保存。
- **R2** 事件运行时 orchestrator 对合法配置行为与迭代前一致（本迭代不改变 DAG 执行语义）。

## 风险与待确认项

- **极旧 DB**：删除结构型 migrate 后，从未被中间版本打开过的备份库可能 **无法启动** 或查询失败；与「配置失效重置」互补，需在发布说明中写明。
- **压缩条件等其它 KKV 文档**：本迭代不改其行为；若需统一为失效+重置，后续单列迭代。
