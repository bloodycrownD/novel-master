# Mobile 稳定性修复与数据库迁移 PRD

## 背景

Android Mobile 在近期使用与发布反馈中出现四类问题：

1. **创建服务商后延迟闪退**：完成服务商创建流程后，应用在一段时间内崩溃，与具体点击哪个按钮无强关联。
2. **聊天消息长按菜单过小屏不可见**：消息长按弹出的操作菜单项较多，在小屏设备上底部项被裁切或无法滚动到，无法完成操作。
3. **覆盖安装与卸载重装行为不一致**：直接覆盖安装 APK 后，聊天气泡等界面出现 DOM 相关异常（如 DOM 不存在）；卸载后重装则正常。
4. **缺少数据库迁移手段**：用户无法在换机或备份场景下导出/导入完整应用数据，现有 VFS ZIP 仅覆盖文件树，不含会话、消息、服务商等 SQLite 数据。
5. **目录规则填充项过多/不适用**：VFS「目录规则」Sheet 中「其余文件填充」仍提供 **全文本**（`fill: full`，即对未命中 head/tail 的文件注入全文）选项，与 Mobile 侧预期不符，需从 UI 移除。

本迭代以 **Android 真机验收** 为主，提升稳定性、小屏可用性与可迁移性，并收敛目录规则配置项。

## 目标（含成功指标）

| 目标 | 成功指标 |
|------|----------|
| 消除服务商创建后的崩溃 | 创建服务商后连续使用应用 ≥10 分钟无崩溃（含返回列表、进入详情、切换 Tab） |
| 小屏可完整使用消息长按菜单 | 屏高 ≤ 640dp（或约定小屏机型）上，长按消息后所有菜单项可访问（滚动或折叠） |
| 安装方式行为一致 | 覆盖安装与卸载重装后，同一会话聊天气泡渲染均正常，无 DOM Exception |
| 支持全库迁移 | 用户可从「我的」导出完整 DB 包，并在新安装/清数据后导入恢复，数据与导出前一致 |
| 目录规则填充项收敛 | 「其余文件填充」仅保留文件名 / 头信息 / 不展示，不再出现全文本（full）选项 |

## 用户与场景

- **用户**：在 Android 手机上日常使用 Novel Master Mobile 的配置与聊天功能的用户。
- **场景 A**：在「我的」中添加 LLM 服务商，保存后继续配置模型或回到聊天。
- **场景 B**：小屏手机上长按消息复制/编辑/隐藏等操作。
- **场景 C**：发布新版本后通过 APK **覆盖安装**升级，不卸载旧版。
- **场景 D**：换机或备份——导出数据库文件，重装 App 后导入恢复全部项目与会话。
- **场景 E**：在会话/项目 VFS 中配置目录规则，仅需 head/tail 与轻量填充策略，不需「全文灌入」类选项。

## 范围

### 包含范围

- Android：**服务商创建后延迟闪退** 的根因修复与回归。
- Android：**聊天消息长按菜单**（`MessageActionMenu` 及关联交互）在小屏上的可访问性（可滚动、最大高度、安全区等，具体实现见后续 SPEC）。
- Android：**覆盖安装** 导致的聊天气泡 DOM 异常排查与修复；保证与清数据/卸载重装后的渲染路径一致或可自动修复脏状态。
- Android：**全量 SQLite 数据库导出/导入**：
  - 导出入口：「我的」页。
  - 导出内容：当前应用使用的完整 Novel Master 数据库（含项目、会话、消息、VFS 元数据、服务商/模型/Agent 等配置）。
  - 导入行为：**完全替换**当前库，导入前二次确认与风险提示。
  - 文件交互：通过系统分享/文件选择器（如 SAF）保存与选取备份文件（格式由 SPEC 定义，如 `.nmbackup` 或 `.db` 单文件）。
- Android：**目录规则 Sheet** 移除「其余文件填充」中的 **全文本**（`fill: full`）选项；保留 **文件名**、**头信息**、**不展示**（与 Core `FillPolicy` 其余值一致）。

### 不包含范围

- iOS 专项验收与修复（本 PRD 以 Android 为准）。
- 数据库 **合并导入**、增量同步、选择性导出子集（后续迭代）。
- 云端自动备份、加密备份密码（除非 SPEC 阶段确认为实现必需）。
- 服务端/桌面端数据库导入导出。
- 非消息类 BottomSheet 菜单的统一改版（除非修复消息菜单时需抽取通用组件并顺带统一，不单独扩 scope）。
- 修改 Core worktree 对 `fill: full` 的语义或 CLI/桌面端目录规则 UI（本迭代仅 Mobile 配置入口移除该选项）。

## 核心需求

1. **服务商创建稳定性**：完成「添加服务商」保存流程后，应用不得因该次创建触发延迟崩溃；需覆盖创建后浏览列表、进入详情、切换「对话/我的」等常见后续操作。
2. **消息长按菜单小屏适配**：长按 assistant/user 消息弹出的菜单在可视区域内可展示全部项；用户无需猜测是否存在被遮挡的选项。
3. **覆盖安装兼容性**：新版本 APK 覆盖旧版安装后，聊天列表与气泡（含富文本/Markdown 渲染路径）正常显示；不得出现因旧渲染缓存、错误 DOM 节点引用导致的崩溃或红屏。
4. **全库导出**：在「我的」提供「导出数据库」；生成单一备份文件，包含当前连接中的全部应用数据；导出成功有明确反馈（路径或分享面板）。
5. **全库导入（替换）**：在「我的」提供「导入数据库」；用户选择备份文件后，经警告确认后替换当前库并重启或重新加载运行时；导入成功后项目/会话/消息与备份一致。
6. **安装与数据边界说明（产品层）**：导入替换会清除当前未备份数据；覆盖安装不应要求用户必须卸载（修复后两种安装方式均应可用）。
7. **目录规则移除全文本填充**：Mobile「目录规则」中「其余文件填充」不得再提供全文本（`full`）；用户只能在新规则或编辑时选择文件名 / 头信息 / 不展示。

## 验收标准

### 1. 服务商创建后闪退（Android）

- **Given** 全新或已有数据的 App，进入「我的」→ 添加服务商  
  **When** 填写必填项并保存成功，之后 10 分钟内执行：返回列表、打开该服务商详情、切换 Tab、发起一次聊天  
  **Then** 应用不崩溃、不 ANR。

- **Given** 连续创建 2 个不同服务商  
  **When** 同上后续操作  
  **Then** 仍无崩溃。

### 2. 消息长按菜单（Android 小屏）

- **Given** 屏高较小设备或开发者选项模拟小屏（如 640×360 dp 档）  
  **When** 在含多条操作项的消息上长按（含 assistant 气泡）  
  **Then** 所有菜单项可通过滚动或等价交互触达，且选中后有预期反馈。

- **Given** 菜单项数量 ≤ 3 的短菜单  
  **When** 长按  
  **Then** 布局正常，无多余空白或裁切。

### 3. 覆盖安装 vs 卸载重装（Android）

- **Given** 旧版 App 已安装且存在至少一条含 Markdown/富文本的 assistant 消息  
  **When** 使用新版 APK **覆盖安装**（不卸载、不清数据）并打开同一会话  
  **Then** 消息列表与气泡正常渲染，Logcat 无未捕获 DOM/React 渲染致命错误。

- **Given** 同上数据  
  **When** **卸载后重装** 同版本 APK（数据已清空则跳过气泡检查，或先导入备份再检查）  
  **Then** 行为与覆盖安装修复后一致（气泡正常）。

- **Given** 覆盖安装后曾出现 DOM 异常的构建  
  **When** 安装含修复的版本  
  **Then** 不再复现「DOM 不存在」类异常（或首次启动自动清理导致异常的缓存状态）。

### 4. 数据库导出（Android）

- **Given** App 内已有项目、会话、消息、服务商配置  
  **When** 在「我的」执行「导出数据库」并完成分享/保存  
  **Then** 生成可再次使用的备份文件；文件大小 > 0；重新安装 App 后可通过导入恢复上述数据。

### 5. 数据库导入 — 完全替换（Android）

- **Given** 设备 B 上为空白或另有无关数据的 App  
  **When** 选择设备 A 导出的合法备份并确认「替换全部数据」  
  **Then** 导入完成后，项目列表、会话、消息、服务商与设备 A 导出前一致（抽样核对：至少 1 个项目、1 会话、1 条消息、1 个服务商）。

- **Given** 导入确认对话框  
  **When** 用户取消  
  **Then** 当前数据不变。

- **Given** 选择非法/损坏文件  
  **When** 导入  
  **Then** 失败提示明确，App 不处于半损坏状态（仍可启动或使用原库）。

### 6. 目录规则 — 移除全文本填充（Android）

- **Given** 打开任意目录的「目录规则」Sheet  
  **When** 查看「其余文件填充」选项  
  **Then** 仅显示 **文件名**、**头信息**、**不展示** 三项，**无「全文本」**（或曾用「全展示」类文案的 full 选项）。

- **Given** 数据库中某目录已保存 `fillPolicy: full`（历史数据）  
  **When** 打开该目录规则 Sheet  
  **Then** 不出现无法理解的空白选中态：应回退显示为约定默认值（如 **不展示** 或 **头信息**，由 SPEC 锁定），保存后不再写回 `full`。

- **Given** 新建或编辑目录规则并保存  
  **When** 在 Sheet 中切换填充策略  
  **Then** 无法选择全文本；保存后 worktree 行为符合所选策略（head/tail + filename/header/hidden）。

### 回归

- **Given** 未使用导入/导出  
  **When** 日常聊天、VFS、Agent 配置  
  **Then** 与修复前正常路径一致，无新增崩溃。

- **Given** VFS 导出 ZIP 既有功能  
  **When** 在会话工作区导出 ZIP  
  **Then** 仍可用，与全库导出职责不冲突。

## 风险与待确认项

| 项 | 说明 |
|----|------|
| 延迟闪退根因未明 | 需日志/复现路径（创建后多久、是否在后台）；PRD 按「创建后 10 分钟内稳定」验收。 |
| 覆盖安装 DOM 异常 | 可能与 `react-native-render-html`、富文本缓存或 MMKV/DB 版本并存有关，SPEC 阶段需定位。 |
| 导入替换需停写数据库 | 导入时可能需要关闭连接并重启 App，需在 UI 上明确提示。 |
| 备份文件安全 | 全库含消息与 API Key 等敏感信息，导出文件应落在用户可控存储，PRD 不强制加密。 |
| 历史 `fill: full` | 仅移除 Mobile 选项；已存 full 的目录需约定展示/迁移策略，避免升级后 Sheet 状态异常。 |
