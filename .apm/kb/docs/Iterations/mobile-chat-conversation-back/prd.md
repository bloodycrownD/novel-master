# Mobile 聊天对话页返回导航 Bugfix PRD

> **类型**：缺陷修复 / 体验修复（bugfix）  
> **平台**：Android（真机 / 模拟器）  
> **路径**：`apps/mobile` → 聊天 Tab → 已进入某会话的**对话页**（消息列表 + 输入框）

## 背景

在 Android 上，用户进入聊天 Tab 内某一会话的对话页后，使用**屏幕边缘侧滑**（系统返回手势）时，应用会直接退出（回到桌面或结束进程），而不是回到同项目下的**会话列表**。

当前顶栏返回按钮已通过 `onBackFromConversation` 回到会话列表，但**系统级返回手势/返回键未接入同一逻辑**，导致与常见即时通讯类 App（对话 → 列表 → 再退出）的操作习惯不一致，影响日常使用。

## 目标（含成功指标）

| 目标 | 成功指标 |
|------|----------|
| 对话页可返回列表 | 在对话页使用边缘侧滑或系统返回键，**100%** 回到当前项目的会话列表，且仍停留在聊天 Tab |
| 不误杀进程 | 对话页单次返回操作**不**直接退出应用 |
| 列表层返回符合习惯 | 已在会话列表时，再侧滑/按返回键可**退出应用**（或回到桌面） |
| 行为一致 | 边缘侧滑、系统返回键、顶栏返回（若展示）在对话页/列表层的行为与上表一致 |
| 不扩功能面 | 不改变会话数据、消息发送、VFS 等业务逻辑 |

## 用户与场景

| 用户 | 场景 |
|------|------|
| 日常写作用户 | 在对话中查看消息后，用手势返回继续选择/切换其他会话 |
| 多会话用户 | 误进某会话后快速返回列表，而非被迫退出整个 App 再重新打开 |

## 范围

### 包含范围

1. Android 上聊天 Tab **对话子视图**（`chatSubview === 'conversation'`）的系统返回处理：边缘侧滑、系统返回键均回到**会话列表**（`chatSubview === 'sessions'`）。
2. 聊天 Tab **会话列表**子视图下，系统返回退出应用（或与 Android 任务栈一致的退出行为）。
3. 与现有 `onBackFromConversation` / 顶栏返回语义对齐，避免同一界面多种返回结果。
4. 手工验收：对话页侧滑、返回键、顶栏返回（如有）及列表层再返回。

### 不包含范围

- iOS（本期未报告；修复后若复现可单独立项验收）。
- 其它 Tab（Agent、我的）及 Stack 子页面（服务商、正则等）的返回栈调整。
- 会话抽屉、批量选择、模板子 Tab 的交互改版（除非与返回栈冲突且为修复必需）。
- Web/桌面端、`examples/mobile` 原型。
- 技术方案、组件拆分、导航库升级（另文 SPEC，本 PRD 不涉及）。

## 核心需求

1. **对话页拦截系统返回**：当用户处于会话对话页时，系统返回（含边缘侧滑）应执行「回到会话列表」，而非将任务栈弹出至退出 App。
2. **列表页放行系统返回**：当用户处于会话列表时，系统返回应允许退出应用（符合用户确认的 Android 习惯）。
3. **与顶栏一致**：对话页顶栏返回（`onBackFromConversation`）与系统返回到达同一界面状态（会话列表、当前项目上下文不变）。
4. **状态一致**：从对话页返回列表后，当前 `projectId` / 列表数据不因返回而丢失；是否清空 `sessionId` 以不破坏列表高亮/再进会话为准（实现细节见 SPEC，本期需求为「回到列表可继续操作」）。
5. **最小改动**：优先在聊天 Tab / 导航上下文内接入返回处理，避免全局改变其它页面返回行为。

## 验收标准

### 对话页 → 会话列表（主路径）

- **Given** 用户已在聊天 Tab，且 `chatSubview` 为对话页（已选中某会话，可见消息区与输入框）  
- **When** 用户使用屏幕边缘侧滑（系统返回手势）一次  
- **Then** 界面回到当前项目的会话列表；应用仍在运行，聊天 Tab 仍为当前 Tab

- **Given** 同上处于对话页  
- **When** 用户按下 Android 系统返回键一次  
- **Then** 结果与侧滑一致：回到会话列表，不退出应用

- **Given** 同上处于对话页，且顶栏展示返回按钮  
- **When** 用户点击顶栏返回  
- **Then** 结果与系统返回一致：回到会话列表

### 会话列表 → 退出应用

- **Given** 用户已在聊天 Tab 的会话列表（未进入对话页，或刚从对话页返回列表）  
- **When** 用户边缘侧滑或按系统返回键一次  
- **Then** 应用退出或回到桌面（符合 Android 常见「根层返回退出」习惯），且不应停留在对话页

### 负向（修复前缺陷不得复现）

- **Given** 用户处于对话页  
- **When** 用户边缘侧滑一次  
- **Then** 不得出现：直接退出应用、跳过会话列表

### 回归（抽测）

- **Given** 用户从会话列表进入对话、发送一条消息、再系统返回列表、再进入另一会话  
- **When** 完成上述流程  
- **Then** 列表与对话切换正常，无卡死、白屏、Tab 丢失
