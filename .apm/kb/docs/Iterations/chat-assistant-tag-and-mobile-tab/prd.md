# 助手伪标签可见性与对话态藏底栏 PRD

## 背景

Mobile 在「富文本消息」开启时，助手气泡会把模型输出中的伪 XML / 未知 HTML 标签（如 `<file>…</file>`、空的 `<xxx></xxx>`）静默剥掉：空标签处正文出现「挖空」，有内容标签则只剩内文、结构标记消失。真机截图中可见「表现为 之间没有文本」「往 写点什么」「多个 并列即可」「、 、」等断层。Desktop 助手侧对同类尖括号通常更接近原文可见，双端观感不一致。

同时，Mobile 底部「对话 / 我的」Tab 挂在整个 Chat Tab 上；会话列表与对话页只是 Tab 内子视图切换，进入对话后底栏仍常显。用户希望聊天沉浸时不再露出下层 Tab，只有回到会话管理（会话列表）时才能切到「我的」。

本需求只修上述两类体验；**不**把 LLM 常见的 `** 文字**`（开标记后空格）做成 Markdown 容错。

与现状关系：

- 富文本能力与开关已由 `chat-rich-render` 立过项；其风险项曾提到「消毒过严导致内容缺失」，但无「伪标签不得挖空」的业务验收。
- 底栏「对话 + 我的」与对话→列表返回分别由 `mobile-agent-nav-refactor`、`mobile-chat-conversation-back` 覆盖；历史仅约定 **Stack 全屏二级页**藏底栏，**未**约定对话子视图藏 MainTabs。

## 目标（含成功指标）

| 目标 | 成功指标 |
|------|----------|
| 助手伪标签不挖空 | Mobile（富文本开）与 Desktop 助手气泡中，未知/伪 XML 标签以**字面量可见**（不被静默删除）；空标签不再留下正文空洞 |
| 双端观感对齐 | 同一条含 `<file>` / `<xxx></xxx>` 等样例的助手消息，两端均能辨认标签字符，无「只剩空格」 |
| 对话沉浸 | Mobile 处于对话子视图时，下层 MainTabs **不可见、不可点** |
| 会话管理可达「我的」 | 回到会话列表后，底栏恢复；可从列表进「我的」 |
| 范围克制 | 不改变消息落库原文；不改 CommonMark 加粗空格规则；工作区 `.md` 预览不强制本迭代验收 |

## 用户与场景

| 用户 | 场景 |
|------|------|
| 开富文本的写作用户 | 阅读助手关于附件/协议格式的说明时，仍能看到文中的 `<tag>`，句子不断档 |
| Desktop 用户 | 与 Mobile 对照同一会话时，助手气泡不会一边可见标签、一边挖空 |
| 日常聊天用户 | 进入某会话对话后界面更沉浸，不会误点底栏「我的」跳出聊天 |
| 需改配置的用户 | 先返回会话列表，再点「我的」进入配置（可接受多一步） |

## 范围

### 包含范围

1. **助手消息伪/未知标签可见性（Mobile + Desktop）**
   - 目标：聊天**助手**消息展示时，非安全白名单内的类 HTML/XML 标签不得以「删除标签壳」的方式造成正文挖空；应保留为用户可读的字面量（或等价可见形式）。
   - Mobile：覆盖「富文本消息」开启后的助手气泡（含落库后与流式升级为 rich 之后的观感一致：不得出现「先看见尖括号、升级后突然消失」的挖空）。
   - Desktop：助手气泡达到与上相同的「不挖空、标签可辨认」验收；与 Mobile 对齐观感。
   - 富文本关闭时：Mobile 助手纯文本路径仍须保证尖括号字符可见（回归，不引入新挖空）。

2. **Mobile 对话态隐藏下层 MainTabs**
   - 当 Chat Tab 处于对话子视图（含「聊天」与「聊天工作区」分段）时，隐藏底部「对话 / 我的」。
   - 回到会话列表（会话管理）后，底栏恢复显示。
   - 进入「我的」的主路径：会话列表 → 底栏「我的」；对话内现有抽屉快捷能力（如切换模型/智能体）保持可用，不强制新增顶栏「设置」入口。

### 不包含范围

1. 对 `** 文字**`（开 `**` 后空白）的 Markdown 容错或自定义 emphasis。
2. 工作区 `.md` / Markdown 文件预览管线的强制改造与验收（可与聊天共用实现，但不作为本 PRD 必过项）。
3. 用户消息气泡改为富文本，或统一「用户也 enrich」的文案/行为漂移治理（除非为修助手挖空所必需的最小连带）。
4. 取消两 Tab 信息架构、把「我的」改到其他全局入口、或把对话提升为独立 Stack 路由（除非实现藏底栏所必需且不改变产品路径语义）。
5. 消息存储内容改写、提示词拼装协议变更、LLM 输出内容审查。
6. Desktop 底部导航（Desktop 无对应 MainTabs）。

## 核心需求（3-7 条）

1. **空伪标签不挖空**：助手原文含空标签（如 `<xxx></xxx>`）时，气泡中须仍能看到标签字面量，前后文不得无故粘连成空洞句。
2. **有内容伪标签保留结构信息**：如 `<file>path</file>`，用户须能识别「这是带标签的片段」，不得只剩无标记的 `path` 而误以为普通词（字面量整段可见即可）。
3. **双端助手一致**：Mobile 与 Desktop 对同一类样例均满足 1–2；不允许一端 escape 可见、一端 discard 挖空。
4. **对话态藏底栏**：Mobile 对话子视图（聊天 / 聊天工作区）不展示 MainTabs；手势与点击均无法在该态直接切到「我的」。
5. **列表态恢复底栏**：从对话返回会话列表后，底栏重新出现，「我的」可进入且能力清单不变。
6. **返回链不回退**：对话→列表的系统返回 / 顶栏返回语义仍遵循 `mobile-chat-conversation-back`；藏底栏不改变该返回结果。

## 验收标准

- [ ] **Given** Mobile「富文本消息」开启，助手消息含 `表现为 <xxx></xxx> 之间没有文本`  
      **When** 打开该会话气泡  
      **Then** 可见 `<xxx></xxx>`（或完整字面量），不出现「表现为 之间」式挖空

- [ ] **Given** Mobile 富文本开，助手消息含带内容的伪标签（如 `<file>notes.md</file>`）  
      **When** 查看气泡  
      **Then** 标签名与内容均可辨认，不只剩无壳的 `notes.md`

- [ ] **Given** Desktop 助手消息含与上相同样例  
      **When** 查看气泡  
      **Then** 同样不挖空、标签可辨认（与 Mobile 观感对齐）

- [ ] **Given** Mobile 富文本关闭  
      **When** 助手消息含尖括号字符  
      **Then** 仍以纯文本可见，无新增挖空

- [ ] **Given** Mobile 已进入某会话对话页（「聊天」或「聊天工作区」）  
      **When** 查看屏幕底部  
      **Then** 「对话 / 我的」底栏不可见；无法从该页直接点到「我的」

- [ ] **Given** 用户从对话返回会话列表  
      **When** 查看底部  
      **Then** 底栏恢复；可点「我的」进入配置菜单

- [ ] **Given** 对话页  
      **When** 使用系统返回或顶栏返回  
      **Then** 仍回到会话列表（不退出 App），行为与现网 conversation-back 一致

- [ ] **Given** 本迭代范围  
      **When** 回归  
      **Then** `** 文字**` 类写法无新增「必须加粗成功」类验收；工作区 `.md` 预览无强制回归项

## 风险与待确认项

- 流式过程中是否允许短暂明文尖括号、仅在 rich 定稿后保证不挖空——以实现最小改动为准，但定稿态必须满足验收。
- Desktop 若现网已对伪标签 escape 可见，本项以「对齐验收」为主，避免为对齐而引入多余行为变化。
- 从「我的」再点回「对话」时，是否保持离开前的 conversation 子视图：保持现网记忆即可，不单独立项。
