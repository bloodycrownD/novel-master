# vfs-zip-directory Feature PRD

## 背景

既有工作区 ZIP（`vfs-zip-io-agent-tool-policy`、`import-export-navigation-fix`）产品默认为：**按 VFS 域（global / project / session）整包导出，导入则域内全量替换**。入口在 Desktop 顶栏「⋯」、Mobile 路径栏 ZIP 图标。

用户需要：

- **目录级**导入/导出 ZIP（覆盖**当前或指定目录子树**，兄弟路径保留）  
- **入口改挂**：与批量 IO 分工清晰  

本 feature **只负责 ZIP**；非 ZIP 的批量导入导出见 `vfs-batch-io`。

## 目标（含成功指标）

| 目标 | 成功指标 |
|------|----------|
| 子树导出 | 对某目录导出 ZIP 后，解压内容对应该目录下文件/空目录树；不含兄弟路径 |
| 子树导入 | 确认导入后，**仅目标目录子树**与 ZIP 一致；域内其他路径**全部保留** |
| 入口正确 | Desktop：目录行（含 `/`）与 blank（≡`/`）均可「导入 ZIP / 导出 ZIP」。Mobile：文件夹内「更多」含「导入 ZIP」；**目录项**菜单含「导出 ZIP」 |
| 确认文案 | 不再暗示「替换整个工作区」，除非目标目录为域根且操作语义即为该子树=`/` |
| 安全 | 取消或不确认 → 子树不变；校验失败 → 不删除已有子树 |

## 用户与场景

| 用户 | 场景 |
|------|------|
| 章节作者 | 只导出/替换 `第3章/`，不影响大纲与其他章 |
| Mobile | 进入某文件夹 → 更多 → 导入 ZIP；长按子文件夹 → 导出 ZIP |
| Desktop | 在资源管理器目录行（含 `/`）或空白区右键导入/导出 ZIP |

## 范围

### 包含范围

1. **导出 ZIP（子树）**：以当前浏览目录或右键/菜单指定的目录为根，打成 ZIP。  
2. **导入 ZIP（子树）**：写入指定目录；先清该子树再写入 ZIP 内容（子树内全量替换）；**不**删除兄弟路径。  
3. **入口**  
   - **Desktop**：**目录行**（含域根 `/`）右键 — 导入 ZIP、导出 ZIP（目标 = 该目录）；**空白区**右键 — 同样提供导入/导出 ZIP，且目标 ≡ `directoryPath:/`。顶栏整域 ZIP 入口**移除**，避免双入口。  
   - **Mobile**：路径栏上方「更多」— **导入 ZIP**（目标 = 当前浏览目录）；**目录项**菜单 — **导出 ZIP**（目标 = 该项目录）。顶栏 ZIP 图标迁走或与菜单二选一，避免双入口。  
4. Desktop / Mobile /（建议）CLI 子树参数与 UI 对齐；仍仅 UTF-8 文本树，与既有 ZIP 约束一致。  
5. 确认对话框文案改为子树覆盖语义。

### 不包含范围

- 非 ZIP 批量导入/导出、拖拽（见 `vfs-batch-io`）  
- 数据库备份、Agent YAML  
- 导入时同步清理/重建 workplace 规则（与既有整域 ZIP 一致：ZIP 不管规则表；若需规则同步另开需求）  
- 二进制文件  
- Mobile 拖放  

## 核心需求

1. **导出范围 = 目录子树**；ZIP 内路径相对该目录（或文档约定的等价相对规则），保证可再导入同一相对位置。  
2. **导入范围 = 目标目录子树覆盖**；兄弟保留。  
3. **域根 `/`**：对域根做子树导入/导出时，行为可等价于「整域文件树」，但文案应说清是「当前目录（根）」，避免与「秘密整域 API」两套心智。  
4. **入口与父级 IA 一致**：更多 vs 目录项职责不混（更多负责导入 ZIP；目录项负责导出 ZIP）。  
5. **相对既有 PRD**：本 feature 将产品默认从「UI 整域全量」调整为「UI 子树」；旧「ZIP = 域真相源」仅在目标为域根时仍成立。  
6. **外部「带域根前缀」ZIP**：导入非域根时若全部 entry 首段 == `basename(directoryPath)` → 失败（`INVALID_PATH`）；**不做自动剥前缀**（算法与正反例见 SPEC）。

## 验收标准

- [ ] Given 域内有 `/a` 与 `/b` 两棵子树，When 对 `/a` 导出 ZIP，Then ZIP 内无 `/b` 下文件  
- [ ] Given 同上，When 对 `/a` 导入仅含新内容的 ZIP 并确认，Then `/a` 与 ZIP 一致且 `/b` **不变**  
- [ ] Given 用户取消确认，When 导入流程结束，Then 目标子树字节级不变  
- [ ] Desktop **目录行（含 `/`）**与**空白区**右键均可见导入/导出 ZIP（blank ≡ `/`）；Mobile 更多可见导入 ZIP，目录项可见导出 ZIP；无互相矛盾的「整工作区替换」误导文案（域根除外已说明）  
- [ ] Given 导入目标为 `/a` 且 ZIP 内全部 entry 首段为 `a/`，When 确认导入，Then `INVALID_PATH`、子树不变，且**无**自动剥前缀写入  

## 风险与待确认项

| 项 | 说明 |
|----|------|
| CLI | 是否增加 `--path` 且默认 `/`；是否废弃无 path 的整域语义 — SPEC 选定：加 `--path`，默认 `/` |
| ZIP 路径兼容 | **已定案（见 SPEC）**：`directoryPath≠/` 且全部 entry 首段 == `basename(directoryPath)` → `INVALID_PATH`；**不做自动剥前缀**。用户须提供相对目标目录的 ZIP |
| 顶栏旧入口 | **已定案**：移除 Header/路径栏整域 ZIP 图标，仅保留目录/blank（Desktop）与更多/目录项（Mobile） |
