# vfs-batch-io Feature PRD

## 背景

用户需要在 ZIP 之外，直接在工作区与本机文件系统之间**批量**搬运文件/文件夹。现状：

- **无**「从本机选多文件/文件夹写入 VFS」或「把 VFS 多选导出到本机目录」。  
- Desktop 工作区树**无**拖拽（无树内移动、无拖入、无拖出）。  
- Mobile「批量操作」= 多选删除 / 目录规则开关，**不是**文件 IO。  
- 父级约定 Mobile **不做拖放**。

本 feature 与 `vfs-zip-directory` 并列，合称四项能力中的「批量导入 / 批量导出」。

## 目标（含成功指标）

| 目标 | 成功指标 |
|------|----------|
| 批量导入 | 用户可将本机多个文件和/或文件夹写入当前（或指定）VFS 目录，保留相对结构 |
| 批量导出 | 用户可将 VFS 中选中的文件/文件夹导出到本机，保留相对结构 |
| Desktop 拖拽三向 | 树内拖动 = 移动；从系统拖入 = 导入；拖出到系统 = 导出 |
| Mobile 菜单 | 「更多」含批量导入、批量导出；可配合既有多选完成导出；无多选时导出当前目录全部 |
| 与 ZIP 分工 | 批量路径不要求用户先打 ZIP；冲突/覆盖有确认或明确策略 |

## 用户与场景

| 用户 | 场景 |
|------|------|
| Desktop | 从资源管理器把一章文件夹拖进会话工作区；或把树里几章拖到桌面；或把 `草稿.md` 拖到 `定稿/` |
| Mobile | 在当前文件夹「更多 → 批量导入」选本机文件/文件夹；多选若干项后「批量导出」到本机；未多选时导出当前目录全部 |

## 范围

### 包含范围

1. **批量导入（非 ZIP）**  
   - 目标目录：当前浏览目录（Mobile 更多）或拖放落点目录（Desktop）。  
   - 支持多文件；**支持文件夹**（递归，保留相对路径）。  
   - 仍以 UTF-8 文本为主；无法作为文本写入的条目须明确跳过或失败提示（不得静默损坏）。  

2. **批量导出（非 ZIP）**  
   - 导出对象：多选的文件/文件夹；**若入口未先多选 → 导出当前浏览目录全部**（UI 须说清）。  
   - 导出到本机目录时保留相对结构。  
   - **Android**：无法选目标文件夹时，降级为多文件/逐文件 `saveDocuments`（`copy: true`）；验收主路径钉死为此，详见 SPEC。  

3. **Desktop 交互（无独立批量按钮）**  
   - **树内拖到另一目录** → VFS 内移动（非导出）。  
   - **从本机拖入树** → 批量导入。  
   - **从树拖出到本机** → 批量导出（`startDrag`；**不以**另存 dialog 作为默认验收替代）。  
   - 三种意图须可区分，不能互相覆盖错。  

4. **Mobile 交互（无拖放）**  
   - **更多菜单**：批量导入、批量导出（可与既有「批量操作」并存，文案区分「文件导入导出」vs「规则/删除」）。  
   - 批量导出可基于多选；无多选 = 当前目录全部；无法选目标文件夹时按 Android `saveDocuments` 降级验收。  

5. Desktop / Mobile（Android）验收；冲突策略产品默认：**覆盖前确认**（SPEC 已钉死）。

### 不包含范围

- ZIP 导入导出（`vfs-zip-directory`）  
- Mobile 拖放  
- 修改既有「批量操作」的删除/规则语义（可并存，勿混名）  
- 二进制大文件策略超出既有 VFS 文本约定的扩展（若做，仅跳过+提示）  
- 云盘同步  
- 以系统分享替代 `saveDocuments` 作为 Android 默认验收主路径（分享可为实现变体，见 SPEC）  

## 核心需求

1. **四项能力中的批量两侧**与 ZIP 入口按父级 IA 划分，文案不互相抢。  
2. **Desktop 三向拖拽均为一等能力**：移动 / 导入 / 导出。  
3. **导入默认合并进目标目录**（不先清空整个域）；与同名冲突须可预期。  
4. **导出保留目录结构**，避免多文件全部摊平到同一层（除非用户只选了单文件）。  
5. **安全**：导入/覆盖需可取消；失败语义见 SPEC — 非 session **整批回滚**，session **逐文件汇总**（`written` / `skipped` / `failed`），禁止无提示半套。

## 验收标准

**Desktop**

- [ ] Given 本机文件夹，When 拖入工作区某目录，Then VFS 下出现对应相对路径的文本文件，兄弟未删  
- [ ] Given 树内文件，When 拖到另一目录，Then 为移动（原路径不存在，新路径存在），不是复制到本机  
- [ ] Given 树内选中项，When **拖出**到系统文件夹（startDrag，非 dialog 另存），Then 本机出现对应文件/目录树；拖出失败有 toast  

**Mobile**

- [ ] Given 当前目录，When「更多 → 批量导入」并完成选择，Then 文件出现在当前目录（或子路径）  
- [ ] Given 多选若干项，When「批量导出」且可选目标目录，Then 本机目录下出现对应相对结构  
- [ ] Given 多选若干项，When「批量导出」且 **Android 无法选目录**，Then 走 `saveDocuments`（多文件或逐文件，`copy: true`），本机可打开导出结果  
- [ ] Given **未多选**，When「更多 → 批量导出」，Then 导出**当前浏览目录全部**（非禁用）  
- [ ] 「更多」中批量导入/导出与「导入 ZIP」并存且文案可区分  

**通用**

- [ ] 用户取消覆盖确认后，目标处原文件保持不变  
- [ ] 非 session 批量写入中途失败 → 整批不落库；session 下部分失败 → UI/结果含已写入与失败项说明  

## 风险与待确认项

| 项 | 说明 |
|----|------|
| Mobile 选目录 API | **已定案**：Android 无法选目录 → `saveDocuments` 降级；无多选 → 导出当前目录全部（见 SPEC） |
| 拖出权限 / 临时文件 | Desktop 拖出须 preload + main `startDrag`；失败 toast；dialog 不作默认验收替代 |
| 与「批量操作」文案 | 避免两个都叫「批量」却含义不同；建议「批量导入/导出」固定指本 feature |
| 空目录 | 导入本机空文件夹是否在 VFS 创建目录节点 — SPEC：是（`mkdir` 显式节点） |
| BatchApplyReport | **已定案**：字段 `written` / `skipped` / `failed`；非 session 整批回滚 vs session 逐文件汇总 |
