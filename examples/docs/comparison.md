# 桌面端 vs 移动端对比

本文档详细对比桌面端和移动端原型的设计差异和实现方式。

## 📐 布局架构

### 移动端（Mobile）
```
┌─────────────────────┐
│  ← 会话    ☰  🌙    │  顶部导航栏
├─────────────────────┤
│                     │
│                     │
│    主内容区域        │  单栏布局
│    （全屏显示）      │  垂直滚动
│                     │
│                     │
├─────────────────────┤
│ 对话 | Agent | 我的 │  底部导航
└─────────────────────┘
```

### 桌面端（Desktop）
```
┌────────┬──────────────────┬────────┐
│ Novel  │   第一卷创作      │ 工作区  │
│ Master │                  │ / 日志  │
├────────┤                  ├────────┤
│ 📱会话  │                  │        │
│ 🤖Agent│   消息列表        │ 文件树  │
│ ⚙️设置 │                  │        │
│        │                  │        │
│ 会话1  │                  │ 日志   │
│ 会话2  │                  │ 记录   │
│ 会话3  │   输入框          │        │
└────────┴──────────────────┴────────┘
  280px        自适应          320px
```

## 🎯 核心差异

| 特性 | 移动端 | 桌面端 |
|------|--------|--------|
| **布局方式** | 单栏垂直 | 三栏水平 |
| **导航位置** | 底部 Tab | 左侧边栏 |
| **项目切换** | 左侧抽屉 | 顶部下拉 |
| **工作区** | Tab 切换 | 右侧固定 |
| **屏幕利用** | 全屏单任务 | 多区域并行 |
| **交互方式** | 触摸优化 | 鼠标优化 |
| **最小宽度** | 320px | 900px |
| **最佳宽度** | 375-428px | 1200px+ |

## 🎨 视觉设计

### 间距和尺寸

| 元素 | 移动端 | 桌面端 |
|------|--------|--------|
| 按钮高度 | 44px（触摸友好） | 36px |
| 输入框高度 | 44px | 36px |
| 卡片内边距 | 16px | 24px |
| 列表项高度 | 最小 44px | 最小 36px |
| 圆角大小 | 12-20px | 8-16px |
| 字体大小 | 15-16px | 14px |

### 颜色系统

两端使用相同的颜色变量，但深色主题略有差异：

**移动端深色主题**
```css
--bg-color: #000000;        /* 纯黑背景 */
--surface-color: #1C1C1E;   /* iOS 风格 */
```

**桌面端深色主题**
```css
--bg-color: #1C1C1E;        /* 深灰背景 */
--surface-color: #2C2C2E;   /* 更柔和 */
```

## 🔄 导航模式

### 移动端：底部导航 + 抽屉
```
主导航：底部 Tab（对话、Agent、我的）
  └─ 对话
      ├─ 会话列表（默认）
      │   ├─ 会话 Tab
      │   └─ 项目模板 Tab
      └─ 聊天界面
          ├─ 聊天 Tab
          └─ 会话工作区 Tab

项目切换：左侧抽屉（☰ 按钮）
会话操作：右侧抽屉（聊天界面中）
```

### 桌面端：侧边栏导航
```
主导航：左侧边栏（会话、Agent、设置）
  ├─ 会话
  │   └─ 会话列表（始终可见）
  ├─ Agent
  │   └─ Agent 列表（始终可见）
  └─ 设置
      └─ 设置菜单（始终可见）

项目切换：左侧顶部下拉
工作区/日志：右侧边栏 Tab
```

## 🎭 页面流程

### 移动端：层级导航
```
会话列表
  └─ 点击会话
      └─ 聊天界面
          ├─ 返回按钮 → 会话列表
          ├─ ☰ 按钮 → 会话操作抽屉
          │   ├─ 切换模型
          │   ├─ 真实提示词
          │   └─ 会话日志
          └─ Tab 切换
              ├─ 聊天
              └─ 会话工作区
```

### 桌面端：并行视图
```
左侧：会话列表（始终可见）
  └─ 点击会话 → 中间区域更新

中间：聊天界面（始终可见）
  ├─ 消息列表
  ├─ 输入框
  └─ 顶部操作
      ├─ 切换模型
      └─ 会话菜单

右侧：工作区/日志（始终可见）
  ├─ 工作区 Tab
  └─ 日志 Tab
```

## 🎯 使用场景

### 移动端适用场景
- 📱 手机浏览器
- 📱 移动应用（React Native）
- 📱 平板竖屏模式
- 📱 小屏幕设备（< 768px）
- 📱 触摸优先交互
- 📱 单手操作

### 桌面端适用场景
- 💻 桌面浏览器
- 💻 Electron 应用
- 💻 平板横屏模式
- 💻 大屏幕设备（> 900px）
- 💻 鼠标键盘交互
- 💻 多任务并行

## 🔄 响应式策略

### 移动端
```css
/* 固定宽度容器 */
#app {
    max-width: 428px;
    margin: 0 auto;
}

/* 针对小屏幕优化 */
@media (max-width: 375px) {
    /* 调整字体和间距 */
}
```

### 桌面端
```css
/* 弹性布局 */
#app {
    display: flex;
    height: 100vh;
}

/* 中等屏幕 */
@media (max-width: 1200px) {
    --sidebar-width: 240px;
    --right-sidebar-width: 280px;
}

/* 小屏幕：隐藏右侧边栏 */
@media (max-width: 900px) {
    #rightSidebar { display: none; }
}
```

## 📝 总结

### 移动端特点
- 🎯 **专注单任务**：一次只做一件事
- 👆 **触摸优先**：大按钮、手势操作
- 📱 **垂直布局**：适合滚动浏览
- 🎨 **简洁界面**：减少视觉负担
- ⚡ **快速切换**：底部导航便捷

### 桌面端特点
- 🎯 **多任务并行**：同时查看多个区域
- 🖱️ **鼠标优先**：悬停、右键、拖拽
- 📐 **水平布局**：充分利用宽屏
- 🎨 **信息密度**：展示更多内容
- ⚡ **快速访问**：侧边栏始终可见

### 选择建议

**使用移动端原型，如果：**
- 目标设备是手机或平板
- 用户主要使用触摸操作
- 需要单手操作支持
- 屏幕宽度 < 768px

**使用桌面端原型，如果：**
- 目标设备是电脑
- 用户主要使用鼠标键盘
- 需要多任务并行
- 屏幕宽度 > 900px

**两者都需要，如果：**
- 构建跨平台应用
- 需要响应式设计
- 用户在多设备间切换

---

**建议**：根据实际使用场景选择合适的原型，或结合两者优点创建响应式设计。
