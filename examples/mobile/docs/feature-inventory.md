# 工作区与协作 — 功能清单（对齐 HTML 原型实现）

> **用途**：RN / Android App 实现的 **UI 与交互权威清单**（以可运行原型为准）。  
> **代码来源**：[`index.html`](../index.html) + [`js/app.js`](../js/app.js)（2026-05 审计）。  
> **说明**：只描述用户可见能力；存储为 mock（localStorage / 内存），正式 App 接 Core。  
> **与旧版差异**：底栏现为 **2 Tab**（对话 | 我的）；Agent 迁入「我的 → 配置」；对齐 `apps/mobile` ProfileTabScreen。

## 1. 全局导航与反馈

### 1.1 信息架构（以实现为准）

```text
底栏: 对话 | 我的
对话 Tab:
  ├─ 当前项目 Banner
  ├─ 子 Tab: 会话 | 项目模板(VFS project)
  ├─ 进入会话后
  │    ├─ 子 Tab: 聊天 | 会话工作区(VFS session)
  │    └─ ☰ → 会话操作抽屉
  └─ ☰(列表态) → 项目抽屉
我的:
  ├─ 工作区: 当前模型/agent/正则组、流式、富文本、Token 计数器
  ├─ 数据管理: 导出/导入数据库 mock
  ├─ 配置: agent管理、服务商、压缩条件、事件配置、正则、全局模板
  └─ 各二级全屏页
```

### 1.2 全屏栈与返回

- `navigateToPage(id, pushToStack)` 维护 `pageStack`
- 返回键弹出栈顶；切换底栏 Tab 时恢复对话子视图状态（`restoreChatSubviewUI`）
- 进入全屏页时关闭抽屉；切换列表批量模式时 `exitBatchMode`

### 1.3 通用反馈

| 能力 | 原型行为 |
|------|----------|
| Toast | `#toast`，成功/提示/错误文案 |
| 确认框 | 原生 `confirm`：批量删除、回滚、离开未保存、删模型/正则组等 |
| 底部 Sheet | 行菜单、VFS 实体菜单、Agent 菜单、添加 Prompt 块类型选择 |
| 模态框 | 见 §0.5 |
| 空状态 | 会话列表、服务商模型列表、正则组列表等文案提示 |

### 1.4 列表批量管理（通用模式）

适用于：**项目**（抽屉内）、**会话**、**Agent**、**服务商**、**组内模型**、**正则组**、**正则规则**。

| 能力 | 说明 |
|------|------|
| 进入管理 | 区头「管理」→ 显示复选框 + 批量头（取消 / 已选 N 项 / 删除） |
| 批量头位置 | 各列表均在 `manage-header`（或项目抽屉头）内切换：取消 / 已选 N 项 / 删除 |
| 选择 | 行复选框；批量模式下点行切换选中，不触发导航 |
| 删除 | 二次确认；项目删除会删其下会话；服务商删除会清其下已保存模型 |
| 提示文案 | 各列表有 `manage-header-hint`（如删服务商同时删已保存模型） |

### 1.5 主题

| 能力 | 说明 |
|------|------|
| 浅色/深色 | `data-theme` on `<html>` |
| 切换 | 顶栏主题按钮 |
| 持久化 | `nm-mobile-theme`（localStorage） |
| 无障碍 | CSS：`prefers-reduced-motion`、`prefers-contrast: high`（见 theme 文档） |

## 4. 虚拟文件（VFS 文件管理器 `vfs-fm`）

三处嵌入，**同一套交互**，仅 `data-vfs-scope` 不同：

| 作用域 | 嵌入位置 | 默认根路径 |
|--------|----------|------------|
| `session` | 对话 → 会话工作区 | `/` |
| `project` | 对话 → 项目模板 | `/template` |
| `global` | 我的 → 全局模板 | `/template` |

### 4.1 列表头（`vfs-fm-header`）

| 控件 | 说明 |
|------|------|
| 返回上级 ↑ | 非根目录可用 |
| 路径 | 单行 `vfs-fm-path`，`title` 为完整路径 |
| 更多 ⋯ | Sheet：新建目录、新建文件、目录纳入规则 |

### 4.2 列表行（`vfs-fm-row`）

| 元素 | 说明 |
|------|------|
| 图标/名称 | 📁 / 📄 |
| 状态副标题 | 目录：`规则·开/关 · N个文件`；文件：`自动/展示/隐藏·显示结果` |
| 徽章 | 文件：隐藏/展示/跟随（显示结果） |
| 规则灯 💡 | 目录且规则开启时高亮 |
| 菜单 ⋮ | 目录/文件不同菜单项 |

### 4.3 交互

| 操作 | 行为 |
|------|------|
| 单击目录行 | 进入子目录 |
| 双击文件行 | 进入 `fileEditor` |
| 目录菜单 | 进入、切换规则开关、目录幻灯片（Toast 示意）、重命名/删除（Toast） |
| 文件菜单 | 打开、切换纳入方式（三态循环）、重命名/删除（Toast） |
| 空目录 | 文案「空目录」 |

**原型未实现**：真实新建/重命名/删除/读写；zip 导入导出；`grep`；阅读模式独立页（仅编辑器内预览按钮 Toast）。

### 4.4 文件编辑页（`fileEditor`）

| 功能 | 原型 |
|------|------|
| 工具栏 | 保存、预览 |
| 源码区 | `textarea` + 静态行号栏（未随滚动同步） |
| 未保存 | 「未保存」指示；离开 `confirm` |
| 保存 | Toast「保存中…」后清除脏标记 |
| 预览 | Toast「切换到预览模式」（无分屏渲染） |

**原型未实现**：检查点下拉回滚、Markdown 渲染视图。

### 4.5 目录纳入规则 Sheet

| 字段 | 控件 |
|------|------|
| sortField | name / ctime / mtime |
| sortDirection | asc / desc |
| headCount / tailCount | range 0–1000 + 实时 output |
| fill | filename / frontmatter / omit |
| 保存 | Toast 反馈；关闭 Sheet |

## 6. 会话日志（统一时间线）

入口：会话操作抽屉 → **会话日志**。

### 6.1 页头

- 警告 Banner：「最多保留 100 个检查点，超出后最旧的将被移除」（示意策略）。

### 6.2 时间线条目

| 类型 | 展示 |
|------|------|
| 工具 · 成功 | 绿条；工具名、时间、摘要；可带检查点链接 |
| 工具 · 失败 | 失败样式；无检查点链接（示例 `replace_in_file`） |
| 工具 · 检查点已移除 | 链接区显示「cp_xxx（已移除）」；回滚按钮 `disabled` |
| 检查点 | ID、时间、来源（工具名或「手动保存」）；「回滚到此」 |

### 6.3 回滚

| 行为 | 原型 |
|------|------|
| 确认 | `confirm` 文案 |
| 进行中 | `rollbackInProgress` → Toast「回滚进行中」 |
| 成功 | Toast「回滚成功」 |
| 从工具条目的检查点链接 | 直接 `performRollback`（不进独立检查点页） |
